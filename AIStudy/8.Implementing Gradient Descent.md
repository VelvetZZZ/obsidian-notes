# 梯度下降的数学实现 (Implementing Gradient Descent)

> [!TIPS] **💡 核心算法公式**
> 梯度下降的本质是一个**不断循环迭代 (Repeat until convergence)** 的过程。在每一次循环中，我们都要用以下两个公式来更新参数 $w$ 和 $b$：
> 
> $$w = w - \alpha \frac{\partial}{\partial w} J(w,b)$$
> $$b = b - \alpha \frac{\partial}{\partial b} J(w,b)$$

## 1. 公式符号彻底拆解

对于刚接触算法的新手，理解公式里的每一个符号至关重要：

* **`=` 的含义 (Assignment vs. Truth assertion)**：
    * 在数学里，$a = a + 1$ 是一个永远不成立的伪命题。
    * 但在代码/算法里，`=` 代表**赋值 (Assignment)**。它的意思是：先计算等号右边的值，然后把这个新值覆盖给左边的变量。
* **$\alpha$ (Alpha)：<mark style="background: #FFB8EBA6;">学习率 (Learning rate)</mark>
    * 它决定了你下山时“迈出的步子有多大”**。步子太小走得慢，步子太大容易跨过谷底。
* **$\frac{\partial}{\partial w} J(w,b)$：<mark style="background: #FFB8EBA6;">导数项 (Derivative)</mark>
    * 它代表了当前位置的“坡度与方向”**。告诉你是该往前走还是往后退，才能让代价 $J$ 变小。

---

##  2. 绝对不能踩的代码陷阱：同步更新 (Simultaneous Update)

这是这张 PPT 里最核心的实操考点。当你用代码实现上述公式时，**必须同时（同步）更新 $w$ 和 $b$**。

### ✅ 正确做法：使用临时变量 (Correct: Simultaneous update)
你需要先算出 $w$ 和 $b$ 的新值，把它们存在“临时变量 (tmp)”里，等两个都算完后，再统一赋值。
```python
# 1. 先用旧的 w 和 b 计算出新的临时值
tmp_w = w - alpha * dJ_dw(w, b) 
tmp_b = b - alpha * dJ_db(w, b) 

# 2. 统一将临时值更新回真实的参数中
w = tmp_w
b = tmp_b
```
## ❌ 错误做法：按顺序更新 (Incorrect)

如果你算出一个就立刻更新一个，会导致严重的逻辑错误。

```Python
# 错误示范！
tmp_w = w - alpha * dJ_dw(w, b)
w = tmp_w  # 此时 w 已经变成了新值！

# 这里的 dJ_db 计算时，使用的是【新的 w】和【旧的 b】
# 这就不是同一个点的梯度了，算法会出错！
tmp_b = b - alpha * dJ_db(w, b) 
b = tmp_b
```

> [!IMPORTANT] **📌 总结**
> 
> 梯度下降的每一步探路，都必须基于你**当前双脚同时站立的那个点**。如果先更新 $w$，就相当于你的一只脚已经迈到了新位置，这时候再用这只脚去测算 $b$ 的坡度，结果当然就不准了。
