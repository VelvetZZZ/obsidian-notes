# CSS 引用方式详解 (Deep Dive)
**日期**: 2026-01-09
**教材章节**: 第3章 CSS样式基础

## 1. 内部样式 (Internal Style)
* **定义**: 将 CSS 代码集中写在 HTML 文档的 `<style>` 标签之间。
* **位置**: 为了结构清晰，一般必须放在 `<head>...</head>` 标签内部。
* **标准语法 (考试背诵)**:
    ```html
    <head>
        <style type="text/css">
            /* 选择器 { 属性: 值; } */
            p {
                width: 150px;
                background: black;
                color: #fff;
            }
        </style>
    </head>
    ```
    * *注意*: 虽然 HTML5 可以省略 `type="text/css"`，但为了应付老教材的填空题，**建议背下来**这个属性写法。

## 2. 内联/行内样式 (Inline Style)
* **定义**: 直接在某个 HTML 标签的 `style` 属性中写 CSS。
* **特点**: 
    * **优先级极高**: 会覆盖掉外部和内部样式表中的同名属性。
    * **作用域小**: 只对当前这**一个**标签生效。
* **缺点**: 书写繁琐，无法实现“结构与表现分离”，**不推荐大量使用**。
* **语法示例**:
 ```html    
 <p style="width:100px; height:40px; color:blue;">我的段落</p>
  ```

### 🎯 考试避坑指南 (Professor's Tips)
1.  **分号陷阱**: 在内联样式 `style="..."` 中，多条声明之间依然要用<span style="background:#fff88f">分号</span>`;` 隔开。
    * 错: `<p style="color:red width:100px">`
    * 对: `<p style="color:red; width:100px;">`
2.  **双引号嵌套**: 如果 HTML 属性用了双引号 `style="..."`，那么里面的字体名称如果有空格（如 "Times New Roman"），建议用**单引号** `'Times New Roman'` 以免冲突。
3.  **分离原则**: 判断题如果问“内联样式是最推荐的编写方式”，一定要打 **❌**。因为它违反了 Web 设计核心的“解耦”原则。


## 3. CSS 引入外部样式 (External Style Sheet)

### 1. 核心定义
* **概念**: 将 CSS 代码单独保存在一个扩展名为 `.css` 的文件中，不包含任何 HTML 标签。
* **作用**: 实现 HTML (结构) 与 CSS (样式) 的**彻底分离**。一个 CSS 文件可以被多个 HTML 页面同时引用，修改一处，所有页面生效。

### 2. 语法格式 (⭐⭐⭐ 必考填空/改错)
必须在 HTML 文档的 `<head>` 标签内部，使用 `<link>`<span style="background:rgba(236, 57, 57, 0.55)"> 单标签</span>进行引入。

```html
<head>
    <link rel="stylesheet" type="text/css" href="css/style.css" />
</head>
```
### 3. 关键属性详解

根据教材和PPT，`<link>` 标签必须指定以下三个属性：

| **属性**   | **值 (Value)** | **说明**                                           | **记忆口诀**                |
| -------- | ------------- | ------------------------------------------------ | ----------------------- |
| **rel**  | `stylesheet`  | 定义当前文档与被链接文档的关系。**必须写**，否则浏览器不知道这是样式表。           | **Rel**ationship (关系)   |
| **type** | `text/css`    | 定义被链接文件的 MIME 类型。虽在 HTML5 中可选，但**考试建议写上**以防扣分。   | 文件类型                    |
| **href** | 文件路径          | 定义 CSS 文件的 URL (相对路径或绝对路径)。例如 `css/ex30303.css`。 | Hyper Reference (超链接引用) |
## 4. 优缺点对比 (多选题考点)

- **优点**:
    
    - 代码复用：多个页面共用一个样式表。
        
    - 维护方便：只需修改一个 `.css` 文件。
        
    - 加载速度：浏览器会缓存 CSS 文件，加快页面加载。
    
- **缺点**:
    
    - 需要额外的 HTTP 请求 (但在现代网络下可忽略)。


## 4. CSS 导入样式 (@import)

### 1. 核心定义
* **概念**: 使用 CSS 提供的 `@import` 规则，在一个样式表中引入另一个样式表。
* **位置**: 必须写在 `<style>` 标签内部，或者另一个 CSS 文件的**最顶部**。

### 2. 语法格式 (两种写法)
根据 PPT，以下两种写法在考试中都算对，但记得它必须包裹在 `<style>` 里：

```html
<style type="text/css">
    /* 写法 A: 使用 url() 函数 (推荐) */
    @import url("css/style.css");
    
    /* 写法 B: 直接使用字符串 */
    @import "css/style.css";
</style>
```
### 3. ⚠️ 铁律 (Exam Rule)

- **顺序至关重要**: <span style="background:rgba(240, 200, 0, 0.2)">`@import` 语句</span>**必须**位于<span style="background:rgba(236, 57, 57, 0.55)">所有其他 CSS 规则之前！</span>
    
- 如果你先写了 `p { color: red; }` 然后再写 `@import`，<span style="background:rgba(240, 107, 5, 0.2)">浏览器会直接**忽略**这个导入语句</span>。

### 4. ⚔️ 考点对比：<link> vs @import

这是**简答题**或**多选题**的高频考点，请务必记下关键词：

| **特性**   | **&lt;link&gt;  标签** | **@import 规则**             | **推荐指数**    |
| -------- | -------------------- | -------------------------- | ----------- |
| **归属**   | HTML 标签              | CSS 语法                     | `<link>` 🏆 |
| **加载顺序** | 页面加载时**同时**加载 CSS    | 页面加载**完后**才加载 CSS (可能导致闪烁) | `<link>` 🏆 |
| **兼容性**  | 无兼容性问题               | 老旧浏览器可能不支持 (CSS2.1+)       | `<link>` 🏆 |
| **JS控制** | JS 可以控制 DOM 修改样式     | JS 无法直接控制                  | `<link>` 🏆 |
>[!TIP] “避坑指南”虽然我们学会了 `@import`，但在**手写代码题**中，除非题目明确要求“请使用导入式”，否则**一律优先使用 `<link>`**。 * **理由**：`@import` 加载慢，容易造成页面“裸奔”（Flash of Unstyled Content），给阅卷老师留下的印象分不好。
