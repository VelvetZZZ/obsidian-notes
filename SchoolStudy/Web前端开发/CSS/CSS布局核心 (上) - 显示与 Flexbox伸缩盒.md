## 1. 盒模型显示类型 (Display Types)

网页标签分为三类，<span style="background:rgba(236, 57, 57, 0.55)">通过 `display` 属性可以互相转换</span>。

| 元素类型                            | 代表标签                            | 特点                                                                              | 转换代码                     |
| :------------------------------ | :------------------------------ | :------------------------------------------------------------------------------ | :----------------------- |
| **块级元素**<br>(Block)             | `div`, `h1-h6`, `p`, `ul`, `li` | 1. **独占一行** (霸道)。<br>2. 宽高、内外边距**都可控**。<br>3. 宽度默认是父容器的 100%。                   | `display: block;`        |
| **行内元素**<br>(Inline)            | `span`, `a`, `b`, `strong`      | 1. **一行显示多个** (并排)。<br>2. **宽高无效** (设置也没用)。<br>3. 只能设置==水平方向==的 margin/padding。 | `display: inline;`       |
| **行内块元素**<br>(==Inline-block==) | `img`, `input`, `td`            | 1. **一行显示多个** (像行内)。<br>2. **宽高可控** (像块级)。<br>3. 是布局中的“完美形态”。                   | `display: inline-block;` |
### ⚠️ 考试易错点：
* **隐藏元素**:
    * `display: none;` -> **彻底消失**，不占位置（后面元素会补上来）。
    * `visibility: hidden;` -> **隐身**，虽然看不见，但**还占着原来的位置**。

## 2. CSS3 伸缩盒布局 (Flexbox) —— ⭐⭐⭐⭐⭐ 必考
教材 9.2.2 章节重点。它是为了解决“元素并排”和“垂直居中”而生的现代布局神器。
### 2.1 核心概念
* **Flex 容器 (Container)**: 父盒子，添加 `display: flex;`。
* **Flex 项目 (Item)**: 子盒子，自动变成伸缩项。
* **两大轴线**:
    * **主轴 (Main Axis)**: 默认是**水平向右** (x轴)。
    * **侧轴 (Cross Axis)**: 默认是**垂直向下** (y轴)。

### 2.2 容器属性 (写在父盒子上)

| 属性名                 | 作用                                    | 常用值 & 含义                                                                                                                                                                                                   |
| :------------------ | :------------------------------------ | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **flex-direction**  | **决定主轴方向**                            | `row` (默认, 水平从左到右) <br> `column` (垂直从上到下, **做手机端布局常用**)                                                                                                                                                    |
| **justify-content** | **主轴**上的对齐方式                          | `flex-start` (左对齐) <br> `flex-end` (右对齐) <br> `center` (居中) <br> `space-between` (**两端贴边**, 中间平分间距) <br> `space-around` (每个盒子周围间距相等)                                                                       |
| **align-items**     | 单行**侧轴**上的对齐方式                        | `flex-start` (顶端对齐)                                                          `flex-end`(底端对齐)<br> `center` (**垂直居中**, 神技!) <br> `stretch` (拉伸, 默认值)                                   `baseline`(项目首行文字对齐) |
| **flex-wrap**       | 是否换行                                  | `nowrap` (默认不换行, 挤在一行) <br> `wrap` (换行)                                                      `wrap-reverse`(行序反向)                                                                                          |
| **align-content**   | 多行项目在侧轴对齐方式（也包括baseline和space-evenly） | 见⬇️笔记                                                                                                                                                                                                      |
### CSS3 Flexbox: 多行对齐 (align-content)

#### 1. 核心定义
* **属性名**: `align-content`
* **作用**: 控制 **多行** 子元素在侧轴（默认是垂直方向）上的排列方式。
* **⚠️ 生效前提 (必考)**: 
    1.  父容器必须开启 Flex (`display: flex;`)。
    2.  **必须允许换行** (`flex-wrap: wrap;`)。
    3.  **必须有多行内容** (如果是单行，这个属性无效，要用 align-items)。
#### 2. 属性值图解 (Visual Guide)
参考你上传的图片，假设侧轴是垂直向下的：

| 属性值 | 含义 | 视觉效果 |
| :--- | :--- | :--- |
| **flex-start** | **起点对齐** | 所有行挤在容器**顶部**，底下留白。 |
| **flex-end** | **终点对齐** | 所有行挤在容器**底部**，顶部留白。 |
| **center** | **居中对齐** | 所有行打包在一起，放在容器**垂直中间**。 |
| **space-between** | **两端对齐** | 第一行贴顶，最后一行贴底，中间行平分间距。 |
| **space-around** | **环绕对齐** | 每一行上下都有相等的空间 (看起来中间的间距比两头大)。 |
| **stretch** | **拉伸** (默认) | 如果子元素没设高度，各行会拉伸填满整个容器高度。 |

#### 3. ⚔️ 终极辨析: align-items vs align-content

| 属性       | `align-items`   | `align-content` |
| :------- | :-------------- | :-------------- |
| **适用场景** | **单行** 或 **多行** | **必须多行** (换行后)  |
| **作用对象** | 每一行里的**单个盒子**   | 作为一个整体的**行**    |
| **比喻**   | 这一排的同学是高还是矮     | 这几排队伍是靠拢还是散开    |
### 2.3 项目属性 (写在子盒子上)
#### 1.  排序属性 (<font color="#c0504d"><span style="background:rgba(236, 57, 57, 0.55)">`order`</span></font>)
默认情况下，Flex 项目按照 HTML 代码的顺序排列。`order` 属性可以改变它们的**视觉顺序**，而不改变 HTML 结构。
* **默认值**: `0`
* **规则**:
    * 数值**越小**，排列**越靠前**。
    * 数值**越大**，排列**越靠后**。
    * <span style="background:rgba(240, 107, 5, 0.2)">支持负值</span>。
* **示例**:
```css
.item-1 { order: 1; }
.item-4 { order: 4; } /* 4号盒子会跑到最后面去 */
```

#### 2. 单独对齐 (<span style="background:rgba(236, 57, 57, 0.55)">`align-self`</span>)
还记得父容器上的 `align-items` 吗？它控制<span style="background:rgba(240, 200, 0, 0.2)">所有孩子的侧轴对齐</span>。如果你想让**某个特定的孩子**“特立独行”，就用 `align-self`。
* **作用**: <span style="background:rgba(240, 167, 216, 0.55)">覆盖父容器的 `align-items` 设置。</span>
* **常用值**: `auto` (默认, 听爸爸的), `flex-start`, `flex-end`, `center`, `stretch`。
* **示例**:
    * 父容器设了 `align-items: flex-start` (都在顶部)。
    * 唯独 3 号盒子设了 `align-self: flex-end` (它自己跑到底部去了)。
#### 3.  弹性核心：Flex 缩放 (Grow, Shrink, Basis)
这是 Flex 布局最核心、也最难懂的三个属性，通常简写为 `flex`。
##### 3.1 放大比例 (`flex-grow`)
* **场景**: 父容器还有**剩余空间** (空白) 时，子盒子怎么<span style="background:rgba(163, 67, 31, 0.2)">瓜分</span>这些空白？
* **默认值**: `0` (不瓜分，给自己多大就是多大)。
* **规则**: 数值代表**份数**。
	    - <span style="background:rgba(240, 167, 216, 0.55)"> 如果所有项目都设为 `1`，则**等分**剩余空间。</span>
	    - 如果一个设为 `2`，其他设为 `1`，则前者抢到的空白是后者的**两倍**。
#### 3.2 收缩比例 (`flex-shrink`)
* **场景**: 父容器**空间不够** (爆了) 时，子盒子怎么<span style="background:rgba(240, 107, 5, 0.2)">缩小</span>自己来挤进去？
* **默认值**: `1` (大家一起等比例缩水)。
* **规则**:
    * 数值**越大**，缩水**越厉害** (越受气)。
    * 数值设为 `0`：**由着性子，绝不缩水** (常用于头像、图标等固定宽度的元素)。
* **示例**:
    * 3 号盒子设了 `flex-shrink: 2`，1号和2号是 `1`。
    * 结果：空间不够时，3 号盒子会被压扁得更惨。
#### 3.3 基础尺寸 (`flex-basis`)
* **作用**: 在分配多余空间之前，项目占据的主轴空间 (类似 `width`)。
* **默认值**: `auto` (也就是看 width 或者是内容本身的大小)。

---
### 4. ⚡ 终极简写 (`flex`) —— 考试必背
我们很少单独写上面三个，而是用简写。
* **语法**: `flex: grow shrink basis;`
* **常见暗号**:
    * **`flex: 1;`**<span style="background:#fdbfff"> (最常用)</span>
        * 等同于: `1 1 0%`
        * 含义: 有空就分(grow=1)，没空就缩(shrink=1)，不在乎自己原本多大。
        * **效果**: 所有 `flex: 1` 的盒子平分父容器的所有空间。
    * **`flex: auto;`**
        * 等同于: `1 1 auto`
        * 含义: 充分利用空间，但会考虑内容原本的大小。
    * **`flex: none;`**
        * 等同于: `0 0 auto`
        * 含义: 既不放大也不缩小，固定尺寸。

## 3.  教授的代码实战 (Code Example)
结合教材内容，我们来实现一个最经典的 **“导航栏布局”** 和 **“完美居中”**。

### 场景 A: 导航栏 (Flex 布局)
让三个菜单项横向排列，并且平均分布。

```html
<style>
    .nav {
        /* 1. 开启 Flex 布局 */
        display: flex;
        /* 2. 主轴对齐：两端贴边，中间平分 */
        justify-content: space-between;
        
        width: 600px;
        height: 50px;
        background-color: #eee;
    }
    .nav span {
        width: 100px;
        height: 40px;
        background-color: skyblue;
        /* 侧轴(垂直)居中 */
        align-items: center; /* 注意：这里其实应该写在父元素 .nav 上，或者单独处理 */
    }
    /* 修正写法：align-items 必须写在容器上 */
    .nav-fixed {
        display: flex;
        justify-content: space-between;
        align-items: center; /* 让子元素在垂直方向居中 */
    }
</style>

<div class="nav-fixed">
    <span>首页</span>
    <span>新闻</span>
    <span>关于我们</span>
</div>
```

### 场景 B: 手机端垂直排列

```css
.box {
    display: flex;
    flex-direction: column; /* 主轴变为垂直方向 */
    align-items: center;    /* 这时侧轴变成了水平方向，这里表示水平居中 */
}
```
