# 📦 包含块 (Containing Block)
**日期**: 2026-01-13
**关联考点**: 习题第 8 题、第 10 题
**核心定义**: 元素在页面中定位和计算大小时的**“基准框”**（参照物）。

## 1. 为什么它很重要？
当你说一个盒子 `width: 50%;` 或者 `left: 10px;` 时，浏览器得知道：
* 是**谁的** 50%？
* 是相对于**谁的**左边 10px？
这个“谁”，就是当前元素的**包含块**。

## 2. 如何寻找包含块？(判断规则)
这是考试的绝对核心，分为三种情况：

### 情况 A: 静态定位 / 相对定位 (Static / Relative)
* **规则**: 包含块就是它的**父元素**的内容区 (Content Box)。
* **理解**: 儿子乖乖待在家里，大小位置都看爸爸的脸色。

### 情况 B: 固定定位 (Fixed)
* **规则**: 包含块是 **浏览器窗口 (Viewport)**。
* **理解**: 它是“钉子户”，只认屏幕窗口，跟父元素完全没关系。

### 情况 C: 绝对定位 (Absolute) —— ⭐⭐⭐ 最复杂，也是第8题的考点
* **规则**: 包含块是 **最近的、一级有定位属性 (position 不是 static) 的祖先元素**。
    * **第一步**: 看爸爸有没有定位？(有 -> 爸爸就是包含块)
    * **第二步**: 爸爸没定位，看爷爷有没有？(有 -> 爷爷就是包含块)
    * **第三步**: 祖宗十八代都没定位？ -> **初始包含块 (浏览器窗口)** 就是包含块。
* **习题第 10 题解析**:
    * A. 包含块的作用是为绝对定位的元素提供定位基准 (✅ 正确)
    * B. 包含块指的是绝对定位的父级容器 (❌ 错，可能是爷爷级)
    * C. 包含块指的是相对定位的父级容器 (❌ 错，祖先也可以是 absolute 或 fixed)

## 3. 回看第 8 题的“破案”过程
结合包含块的概念，我们再看一眼刚刚那道坑题：

* **子元素 #b**: `position: absolute`。它在寻找它的**包含块**。
* **父元素 #a**: `float: right` (position 默认为 static)。
* **判断**: 因为父元素是 static，所以**父元素无法成为包含块**。
* **结果**: #b 继续向上找，最终找到了**初始包含块 (浏览器窗口)**。
* **结论**: 所以偏移是相对于浏览器窗口左上角的。
