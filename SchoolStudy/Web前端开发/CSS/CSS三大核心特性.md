# CSS 三大核心特性 (The Big Three)

**教材章节**: 第3章 CSS样式基础
**考点**: 哪些属性可继承、权重计算公式 (a,b,c)、!important 的作用

## 1. 🧬 继承性 (Inheritance)
* **定义**: 子元素会自动拥有父元素设置的**某些**样式属性。
* **规则**:
    * ✅ <span style="background:rgba(205, 244, 105, 0.55)">可继承</span>: 主要是**文本类**属性。
        * `text-` 开头 (如 `text-align`)
        * `font-` 开头 (如 `font-size`, `font-family`)
        * `line-` 开头 (如 `line-height`)
        * `color` (字体颜色)
    * **❌<span style="background:rgba(240, 107, 5, 0.2)"> 不可继承</span>**: 主要是**盒子模型/布局类**属性。
        * `border`, `padding`, `margin`, `width`, `height`, `background` 等。
* **特例**: `<a>` 标签通常不会继承父元素的颜色（因为<span style="background:rgba(240, 200, 0, 0.2)">浏览器有默认样式覆盖</span>），必须直接选中它来修改。

## 2. 📚 层叠性 (Cascading)
* **定义**: 当**权重相同**的多个选择器选中同一个元素，且设置了冲突的属性时，CSS 如何解决冲突？
* **原则**: **就近原则 (Last man standing)**。
    * 哪个样式写在代码的**最后面**（<span style="background:rgba(240, 107, 5, 0.2)">最下面</span>），就执行哪个。
    * *口诀*: <span style="background:#fff88f">后来者居上</span>。

## 3. ⚖️ 优先级与权重 (Priority & Specificity) —— ⭐⭐⭐⭐⭐ 核心考点
当**权重不同**的选择器选中同一个元素时，谁的权重高，谁说了算。

### 3.1 基础权重金字塔
(从<span style="background:rgba(236, 57, 57, 0.55)">高到低</span>排列)
1.  **!important**: 核武器，权重最高，无视一切规则。
2.  **行内样式** (`style="..."`): 权重极高。
3.  **ID 选择器** (`#id`): 权重高。
4.  **类 / 伪类 / 属性选择器** (`.class`, `:hover`): 权重中。
5.  **标签 / 伪元素选择器** (`div`, `p`): 权重低。
6.  **通配符** (`*`): 权重最低 (0)。

### 3.2 🔢 权重计算公式 (a, b, c) —— 必考计算题
对于复杂的<span style="background:rgba(236, 57, 57, 0.55)">复合选择器</span>，我们需要通过计算**权重值**来比较。
我们将权重表示为三个数字的组合：**(a, b, c)**

| 位数 | 代表含义 | 计数规则 |
| :--- | :--- | :--- |
| **a** | **ID 选择器**的个数 | 代码中出现了几个 `#` |
| **b** | **类、伪类**的个数 | 代码中出现了几个 `.` 或 `:` |
| **c** | **标签**的个数 | 代码中出现了几个标签名 (如 `div`, `p`) |

* **比较规则**:
    1.  先看 **a** (ID数)：谁的 a 大，谁就赢（ID多者胜）。
    2.  如果 **a** 相同，看 **b** (类数)：谁的 b 大，谁就赢。
    3.  如果 **b** 相同，看 **c** (标签数)：谁的 c 大，谁就赢。
    4.  如果 **(a, b, c)** 全都一样，则遵循**层叠性**（谁写在后面谁赢）。

### 3.3 💡 教授的计算演示
假设有以下 HTML:
```html
<div id="box" class="container">
    <p class="text">你好</p>
</div>
```
比较以下两个选择器的权重：

- **选择器 A**: `#box .text`
    
    - ID = 1 (`#box`)
        
    - Class = 1 (`.text`)
        
    - Tag = 0
        
    - **权重**: (1, 1, 0)
        
- **选择器 B**: `div .container p`
    
    - ID = 0
        
    - Class = 1 (`.container`)
        
    - Tag = 2 (`div`, `p`)
        
    - **权重**: (0, 1, 2)
**结果**: <span style="background:rgba(236, 57, 57, 0.55)">(1, 1, 0) > (0, 1, 2)。因为 A 有 ID，而 B 没有，**ID 是王炸**，直接胜出，不需要看后面的位数。</span>

>[!CAUTION] 教授的“避坑指南” (Professor's Tips)
在做期末考试的**计算题**时，有几个陷阱你需要特别注意：   
> 1. **不进位原则**: 这一点 PPT 没细说，但很重要。10个类选择器 (`b=10`) 的权重，依然**比不上** 1个 ID 选择器 (`a=1`)。不要把类加起来去顶替 ID。 * `(0, 11, 0)` < `(1, 0, 0)` 
>2. **继承的权重为 0**: 如果一个样式是“继承”自父亲的（比如父亲设了红色，儿子没设），那么它的权重是最低的。哪怕你给父亲加了 `!important`，只要儿子自己有一个哪怕是标签选择器 (`p { color: blue }`)，儿子都会听自己的，而不是继承父亲的。 
>3. **!important 的副作用**: 虽然它好用，但考试时如果问“是否推荐大量使用”，一定要答**否**。因为它会破坏 CSS 的层叠规则，导致调试极其困难。
