# HTML5 表单设计 (Forms)

## 1. 表单容器 (`<form>`)
**核心作用**：它是所有表单控件（输入框、按钮等）的容器，负责将收集到的数据提交给服务器。

### 1.1 基本语法
```html
<form action="处理程序的URL" method="提交方式" name="表单名称">
    </form>
```
### 1.2 关键属性 (Exam Focus)

这张表格是**高频考点**，特别是 `method` 的取值区别。

| **属性**       | **说明**           | **重点/备注**                            |
| ------------ | ---------------- | ------------------------------------ |
| **`action`** | 指定表单提交的**目标地址**。 | **必有属性**。默认为当前页面。                    |
| **`method`** | 指定数据提交的 HTTP 方法。 | **必有属性**。默认值为 `get`。                 |
| `name`       | 表单名称。            | 以此区分页面中的多个表单，常与 JS 配合使用。             |
| `enctype`    | 表单数据的编码类型。       | 上传文件时必须修改此属性（虽PPT未详述，但需留意）。          |
| `target`     | 提交后在哪里打开结果页面。    | 同链接的 `target` (`_blank`, `_self` 等)。 |

## 1.3 表单核心：GET 与 POST 方法详解

#### 1. 核心对比 (Exam Focus)
这是考试中最爱考的简答题或选择题。

| 维度 | GET 方法 | POST 方法 |
| :--- | :--- | :--- |
| **传输方式** | 通过 URL 传递。格式为 `URL?变量名=值&变量名=值`。 | 数据放在 HTTP 请求体 (Body) 中。 |
| **安全性** | **极低**。数据直接暴露在地址栏中。 | **较高**。操作对用户不可见 (地址栏无变化)。 |
| **数据容量** | **小**。受浏览器地址栏长度限制。 | **大**。适合上传文件或长文本。 |
| **适用场景** | 搜索查询 (非敏感数据)。 | 密码登录、注册、敏感信息提交、文件上传。 |

#### 2. 表单属性配置 (`<form>`)

```html
<form action="target.php" method="post" name="loginForm">
   </form>
```
- **`action`**: 指定表单数据提交的目标位置 (URL)。
    
- **`method`**: 指定提交方式，默认为 `get`。**必选属性**。
    
- **`name`**: 表单名称，常用于 JS 控制。
#### 3. 关键细节与“坑点”

##### 3.1 URL 拼接规则 (GET)

当使用 GET 时，浏览器会自动处理 URL：

1. **`?`**: 连接目标 URL 和参数。
    
2. **`&`**: 连接多个变量。
    
    - _示例_: `index.jsp?user=zhangsan&age=18`
        

##### 3.2 数据的“钥匙”：Name 属性

- **规则**: 表单控件（如 `<input>`）必须有 `name` 属性，数据才能被提交。
    
- **现象**: 如果没有 `name`，提交后地址栏或请求体中不会包含该字段的数据。
    

##### 3.3 密码安全

- 虽然 `<input type="password">` 可以在页面上隐藏输入内容（显示为圆点），但如果配合 `method="get"`，密码依然会**明文**显示在地址栏中。因此密码表单**必须**使用 POST。

## 2. 基础输入控件 (`<input>`)

`<input>` 是单标签，通过 `type` 属性的变化来变身成不同的控件。

### 2.1 文本域 (`type="text"`)

作用：用于输入单行普通文本（如用户名、搜索关键词）。

代码示例：

```html
<input type="text" name="user" value="默认值" />
```

- `value`：设置文本框里的默认文字。
    
- `name`：提交给服务器时的变量名（例如 `user=张三`）。


### 2.2 密码域 (`type="password"`)

作用：用于输入密码。

特点：输入的内容会显示为实心圆点 (●) 或星号，防止旁人偷窥。

代码示例：

```html
<input type="password" name="pwd" />
```

>[!TIP] **⚠️ 教授提醒**：虽然密码域在页面上看不见明文，但如果用了 `GET` 方式提交，密码还是会显示在地址栏里！所以密码表单**必须**用 `POST`。

### 2.3 隐藏域 (`type="hidden"`)

作用：存放需要提交给服务器、但不需要用户看到或修改的数据（如用户ID、Session令牌）。

特点：在页面渲染时不可见，但在“审查元素”代码中可见。

代码示例：

```html
<input type="hidden" name="userId" value="10086" />
```

## 3. 选择类控件 (Selection Controls)
当用户不需要输入文字，而是从选项中进行选择时，我们需要用到以下两种控件。

### 3.1 单选按钮 (`type="radio"`)
**适用场景**：多选一（互斥），例如：性别（男/女）、学历、专业等。

**语法格式**：
```html
<input type="radio" name="分组名称" value="提交的值" /> 选项文字
```
**🔥 考试核心考点 (CRITICAL)**：

1. **分组机制**：多个 radio 按钮要想实现“互斥”（即选了A就不能选B），它们的 **`name` 属性值必须完全相同**。
    
    - _错误示范_：`name="sex1"` 和 `name="sex2"` —— 这样两个都能同时选中。
        
    - _正确示范_：都写 `name="sex"`。
        
2. **默认选中**：使用 **`checked`** 属性（或 `checked="checked"`）来设置默认选中的项。

**代码示例**：

```html
<input type="radio" name="speciality" value="security" /> 信息安全
<input type="radio" name="speciality" value="software" /> 软件工程
<input type="radio" name="speciality" value="comApp" checked /> 计算机应用
```

### 3.2 复选框 (`type="checkbox"`)

**适用场景**：多选多，例如：兴趣爱好、技能掌握等。

**语法格式**：

```html
<input type="checkbox" name="名称" value="提交的值" /> 选项文字
```

**特点**：

- 选项之间互不影响，可以全选，也可以全不选。
    
- 同样可以使用 `checked` 属性设置默认选中状态。


**代码示例** (爱好选择)：


```html
<p>爱好：</p>
<input type="checkbox" name="hobby" value="walk" /> 散步
<input type="checkbox" name="hobby" value="music" checked /> 音乐
<input type="checkbox" name="hobby" value="swim" /> 游泳
<input type="checkbox" name="hobby" value="video" checked /> 视频
```

## 4. 按钮控件 (Buttons)

表单填完了，总得有动作。HTML 提供了三种核心按钮。

|**按钮类型**|**type 值**|**功能描述**|**备注**|
|---|---|---|---|
|**提交按钮**|`submit`|**核心**。将表单数据发送给 `action` 指定的服务器地址。|相当于“发送”键。|
|**重置按钮**|`reset`|**清空**。将表单内所有控件恢复到**页面刚加载时**的初始状态。|相当于“一键撤销”。|
|**普通按钮**|`button`|**无默认行为**。通常配合 JavaScript 脚本使用，触发特定逻辑。|考试中常配合 `onclick` 事件出现。|

**代码示例**：
```html
<form action="index.jsp">
    <input type="submit" value="确认提交" />
    
    <input type="reset" value="重置信息" />
    
    <input type="button" value="点我" onclick="alert('Hello!')" />
</form>
```

## 5. 综合细节与 HTML5 新特性

在 PPT 的综合示例中，还出现了一些 HTML5 新增的便捷输入类型，以及一个极其重要的“丢分坑”。

### 5.1 HTML5 新增类型 (见示例图)

虽然 PPT 没有单独列定义页，但示例代码中出现了，考试可能会涉及：

- **`type="number"`**：数字输入框。
    
    - 常用属性：`min` (最小值), `max` (最大值), `step` (步长)。
        
    - _示例_：`<input type="number" begin="2" end="10" ...>` (注：标准HTML5属性是 `min`/`max`，PPT示例中用了 `begin`/`end`，建议考试时优先确认教材标准，若教材用 `begin` 则跟随教材，通用标准是 `min/max`)。
    
- **`type="date"`**：日期选择器（自带日历控件）。


###  5.2 教授的警示：Name 属性的重要性

**考点**：如果一个表单控件（比如文本框）**没有设置 `name` 属性**，那么在点击提交按钮后，这个控件的数据**不会**被发送给服务器！

- **现象**：地址栏的 URL 中看不到这个参数。
    
- **结论**：想提交数据，`name` 属性必不可少。


```Markdown
工龄：<input type="text" value="5" /> 

工龄：<input type="text" name="workAge" value="5" />
```
