# HTML5 新增语义化标签

**标签：** #Web基础 #HTML5 #语义化 #新特性 #重点
**核心概念：** 用有意义的标签代替无意义的 `<div>`。

---

## 1.  结构布局类 (Layout)
*考试应用：网页布局代码题，用这些标签替代 div。*

| 标签 | 含义 | 作用与记忆 |
| :--- | :--- | :--- |
| **`<header>`** | 页眉 | 网页或版块的头部（Logo、搜索框）。<br>⚠️ *区分：`<head>` 是网页元数据，`<header>` 是可见的头部。* |
| **`<footer`>** | 页脚 | 网页底部（版权信息、备案号）。 |
| **`<nav>`** | 导航 | 存放链接组 (Navigation)。 |
| **`<article>`** | 文章 | 独立的、可复用的内容（如一篇博客）。 |
| **`<aside>`** | 侧边栏 | 和主要内容稍微有点关系，但能独立拆分的内容（如广告、侧栏）。 |
| **`<section>`** | 区域 | 通用的文档章节。 |
| **`<hgroup>`** | 标题组 | 组合 h1-h6 标题（*注：现代标准中争议较大，但教材有*）。 |

---

## 2. 多媒体与绘图 (Multimedia)
*考试应用：选择题、填空题。*

| 标签 | 含义 | 作用 |
| :--- | :--- | :--- |
| **`<video>`** | 视频 | HTML5 原生视频播放器。 |
| **`<audio>`** | 音频 | HTML5 原生音频播放器。 |
| **`<source>`** | 资源 | 为 video/audio 指定文件源。 |
| **`<canvas>`** | 画布 | **考点：** 它是通过 **JavaScript** 在网页上绘制图形（图表、游戏）的容器。它自己不会画画。 |
| **`<embed>`** | 嵌入 | 在 HTML5 中被标准化，用于外部插件。 |
| **`<figure>`** | 图文 | 包含图片 `<img>` 和标题 `<figcaption>` 的容器。 |

---

## 3. 表单与交互 (Forms & UI)
*考试应用：PPT中红线划过的重点。*

| 标签 | 含义 | 作用 (★必背) |
| :--- | :--- | :--- |
| **`<datalist>`** | 数据列表 | 给 `<input>` 提供“自动完成”的下拉建议选项。 |
| **`<mark>`** | 标记 | 黄色高亮显示文本（像荧光笔）。 |
| **`<meter>`** | 度量衡 | 显示已知范围内的标量测量值（如磁盘使用率）。 |
| **`<progress>`** | 进度条 | 显示任务进度（如下载进度）。 |
| **`<details>`** | 详情 | 创建一个可展开/折叠的小部件。 |
| **`<output>`** | 输出 | 专门用于显示脚本计算的结果。 |

## 4. 考古专用 (教材特供)
*教授提示：以下标签在现代 Web 开发中可能已废弃或极少使用，但**教材表中有**，考试需注意。*

* **`<command>`**: 定义命令按钮（现代通常用 button）。
* **`<keygen>`**: 生成密钥对（**已废弃**，浏览器已不支持，但考试别空着）。
* **`<dialog>`**: 对话框（最近浏览器又支持了，用于做模态窗口）。

## 5. ⚡️ 教授的“一句话辨析”

> [!QUESTION] `<section>` 和 `<div>` 有什么区别？
> * `<div>` 是无意义的容器（为了布局而包起来）。
> * `<section>` 是有语义的章节（里面通常应该包含一个标题）。

> [!QUESTION] `<article>` 和 `<section>` 怎么分？
> * `<article>` 是一篇完整的文章，拿走也能独立阅读（如新闻）。
> * `<section>` 是文章里的一个章节（如“第一章”、“第二章”）。
