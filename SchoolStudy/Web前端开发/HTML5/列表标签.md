# HTML5 列表标签复习

## 1. 无序列表 (Unordered Lists)
**适用场景**：列表项之间没有先后顺序之分（如导航菜单、新闻列表）。
**基本标签**：`<ul>` 包裹 `<li>`。

### 1.1 语法结构
```html
<ul>
    <li>列表内容1</li>
    <li>列表内容2</li>
</ul>
```
- `<ul>` (Unordered List): 声明这是一个无序列表。
    
- `<li>` (List Item): 定义具体的列表项。
    

### 1.2 关键属性 `type`

用于定义列表项前面的“小圆点”样式。

|**属性值**|**描述**|**形状示例**|
|---|---|---|
|`disc`|**默认值**。实心圆。|●|
|`square`|实心方块。|■|
|`circle`|空心圆。|○|
|`none`|无符号（常用于去除默认样式）。|(空白)|

## 2. 有序列表 (Ordered Lists)

适用场景：列表项有明确的顺序或排名（如排行榜、步骤说明）。

基本标签：`<ol>`包裹 `<li>`。

### 2.1 语法结构

HTML

```
<ol>
    <li>第一步</li>
    <li>第二步</li>
</ol>
```

### 2.2 关键属性 (考试重点)

1. **`type` 属性**：决定序号的样式。
    
    - `1`：数字 (1, 2, 3...) —— **默认值**。
        
    - `a`：小写字母 (a, b, c...)。
        
    - `A`：大写字母 (A, B, C...)。
        
    - `i`：小写罗马数字 (i, ii, iii...)。
        
    - `I`：大写罗马数字 (I, II, III...)。
        
2. **`start` 属性**：决定从第几个开始计数。
    
    - **注意**：无论 `type` 是什么，`start` 的值永远是**数字**。
        
    - _例如_：`type="a" start="3"`，表示从第3个字母（即 c）开始排序。
        

**代码示例** (参考 PPT 示例 2-8-2)：

HTML

```
<ol type="a" start="3">
    <li>运动</li>
    <li>听音乐</li>
</ol>
```

## 3. 定义列表 (Definition Lists)

适用场景：用于名词解释、问答列表（Q&A）等“标题-内容”成对出现的情况。

结构特点：不同于前两种的简单嵌套，它有三层标签。

### 3.1 语法结构

HTML

```html
<dl>
    <dt>名词 (Term)</dt>
    <dd>描述/解释 (Description)</dd>
    
    <dt>CPU</dt>
    <dd>中央处理器，计算机的大脑...</dd>
</dl>
```

- `<dl>` (Definition List): 定义列表的容器。
    
- `<dt>` (Definition Term): 定义**项目/名字**。
    
- `<dd>` (Definition Description): 定义**描述/内容**（通常会自动缩进）。

 教授的小提示 (Exam Tips)**：
> 
> 1. 块级元素**：PPT 特别标注了，`<ul>`, `<ol>`, `<dl>`, `<li>`, `<dt>`, `<dd>` 全都是**块标签 (Block Element)**，意味着它们默认都会独占一行。
>     
> 2. **有序列表的陷阱**：如果考题让你写代码实现“从字母 D 开始排序”，一定要写 `<ol type="A" start="4">`。千万别写成 `start="D"`，这是错的！`start` 只能是数字。
>     
> 3. **嵌套关系**：`<dl>` 里面只能放 `<dt>` 和 `<dd>`，不要把 `<p>` 或者其他标签直接放在 `<dl>` 的第一层。
>
