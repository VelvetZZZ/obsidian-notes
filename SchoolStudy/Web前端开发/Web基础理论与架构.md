## 1. Web 体系结构与协议基础

这部分主要考查**单选题**和**填空题**，重点在于区分各种协议和端口。

### 1.1 核心概念
* **Web (World Wide Web)**: 万维网。
    * 核心机制：基于 **HTTP** 协议，在 **服务器 (Server)** 和 **客户端 (Client)** 之间传输数据。
    * 架构模式：采用 **B/S (Browser/Server)** 结构，这是一种特殊的 C/S (Client/Server) 结构。
* **超文本 (Hypertext)**:
    * 定义：将各种信息节点（文字、图片、视频）连接在一起的**网状逻辑结构**。
    * 核心：由 **节点**、**链**、**网络** 三要素组成。

### 1.2 常用网络协议 (Protocol)

| 协议缩写       | 全称/含义                         | 端口号 (★必背) | 作用                                |
| :--------- | :---------------------------- | :-------- | :-------------------------------- |
| **HTTP**   | HyperText Transfer Protocol   | <font color="#ff0000">**80**    | **超文本传输协议**。Web通信的基础，无状</font>态，请求/响应模式。 |
| **FTP**    | File Transfer Protocol        | 21        | 文件传输协议（用于上传/下载文件）。                |
| **Telnet** | -                             | 23        | 远程登录协议。                           |
| **SMTP**   | Simple Mail Transfer Protocol | 25        | 简单邮件传输协议（发邮件）。                    |
| **DNS**    | Domain Name System            | 53        | **域名系统**。将域名解析为 IP 地址。            |
| **TCP/IP** | -                             | -         | 传输控制协议/因特网互联协议。互联网的基石。            |

> [!TIP] 协议分层记忆
> * **IP (网络层)**: "快递员"。负责把数据包送到正确的目的地（寻址）。
> * **TCP (传输层)**: "打包员"。运行在 IP 之上，负责数据传输的**可靠性**（不丢包）。

## 2. URL、域名与 IP 地址

这部分是**概念辨析题**的重灾区。

### 2.1 核心术语辨析
* **IP 地址**:
    * 互联网上每台设备的**唯一数字标识**（身份证号）。
    * 用于定位和通信。
* **域名 (Domain Name)**:
    * IP 地址的**人类可读别名**（比如 `tsinghua.edu.cn`）。
    * 方便记忆，背后通过 DNS 服务器指向 IP。
* **URL (统一资源定位符)**:
    * 定义：互联网上任意一个资源（网页、视频、图片）的**完整、唯一**的详细地址。
    * **组成公式**：`协议://主机名(域名):端口/路径/文件名`
    * *示例：* `http://www.tsinghua.edu.cn`

> [!QUESTION] 考试真题模拟
> **Q:** 下面正确的 URL 地址是？
> **A:** `http://www.tsinghua.edu.cn` (注意要有 `http://`)

## 3. Web 标准 (The Web Standards)

这部分考查**网页构成的三要素**以及**为什么要用标准**。

### 3.1 网页三层结构 (★ 重点)
网页由三个部分组成，各司其职：

| 层面                                                                                          | 对应技术                                 | 英文                  | 作用           | 比喻       |
| :------------------------------------------------------------------------------------------ | :----------------------------------- | :------------------ | :----------- | :------- |
| <font color="#d83931">**结构<font color="#000000">标准**                                        | <font color="#ff0000">**HTML**       | Structure           | 结构化网页内容（骨架）。 | 盖房子的钢筋水泥 |
| <font color="#d83931">**表现<font color="#000000">标准</font>**                         </font> | <font color="#ff0000">**CSS**        | Pre</font>sentation | 样式、布局、美化网页。  | 房子的装修风格  |
| **<font color="#d83931">行为<font color="#000000">标准</font>**                         </font> | <font color="#ff0000">**JavaScript** | Behavior  </font>   | 网页的交互、逻辑控制。  | 房子的智能家电  |

### 3.2 为什么要采用 Web 标准？
*(简答题背诵点)*
1.  **兼容性好**：确保不同浏览器显示一致。
2.  **可维护性高**：结构与表现分离，改样式不用动代码。
3.  **利于 SEO**：搜索引擎优化，更容易被搜到。
4.  **提升用户体验**：加载更快，访问更流畅。

> [!WARNING] 易错判断题
> * **目前的 Web 标准包括动态网页吗？** -> **不包括！** (Web标准主要指前端的 HTML/CSS/JS，不包含 PHP/Java 等后端动态技术)。
> * **什么是标准浏览器？** -> 指对 W3C 规范提供完善支持的浏览器（如 Chrome, Firefox, Edge），而不是老旧的 IE。

## 4. HTML5 基础语法与规范

### 4.1 基础定义
* **HTML**: HyperText Markup Language (超文本标记语言)。
* **本质**: 它是一种**标记语言 (ML)**，不是编程语言。由 Web 浏览器解释执行。
* **SGML**: 标准通用标记语言，是 HTML 的“祖先”。

### 4.2 HTML5 语法规则 (★ 改错题/填空题)
1.  **文档类型声明**: 必须在文档开头第一行写 `<!DOCTYPE html>` (不区分大小写)，用于<span style="background:#fff88f">定义文档类型</span>。
2.  **标签嵌套**: 元素**不允许交叉嵌套**。
    * ✅ 正确: `<div><p>内容</p></div>`
    * ❌ 错误: `<div><p>内容</div></p>`
3.  **属性值**: 建议用**双引号** `""` 括起来。
    * 例: `<input type="text">`
4.  **空标签闭合**: 对于单标签，最好加 `/` 关闭（虽然 HTML5 不强制，但 XHTML 习惯更好）。
    * 例: `<br />`, `<hr />`, `<img src="..." />`
5. **属性最好有值**：不能简写。
	- 例:`<input checked="checked" />
6. **注释 (Comments)**: 
   * **语法**: <!-- 这里是注释内容 -->``
   * **作用**: 解释代码逻辑，或者暂时“屏蔽”某段代码（Debug用）。
   * **注意**: 千万不要**嵌套**使用注释！

### 4.3 经典代码辨析 (笔记真题)

> [!EXAMPLE] 判断下列代码是否符合规范
> 1. `<input type="text" disabled="disabled">`
>    * **评价**: ✅ 符合 XHTML/HTML5 严谨规范（属性名=属性值）。
> 2. 下列代码完全符合 HTML5 语法标准的是？
>    * A. `<br>` (HTML5允许，但不如C严谨)
>    * B. `<p>这是一个段落` (HTML5允许省略结束符，但易乱)
>    * C. `<div></div>` (✅ **标准写法**)
>    * D. `<hr>`
>    * **结论**: 考试中选最严谨的 **C**。

## 5. 补充知识点 (笔记细节)

* **MIME**: 多用途互联网邮件扩展类型（告诉浏览器发送的是什么格式的文件，如 `text/html`, `image/jpeg`）。
* **Web 工作流程**:
    1.  客户端发起请求 (Request)。
    2.  DNS 解析域名。
    3.  建立 TCP 连接 (三次握手)。
    4.  服务器处理并返回响应 (Response)。
    5.  浏览器解析 HTML 显示页面。
