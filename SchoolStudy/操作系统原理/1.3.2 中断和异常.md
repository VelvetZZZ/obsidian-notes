# 中断和异常 (1.3.2)

> [!TIP] 核心逻辑：中断的作用
> 1. **夺权**：“中断”是让操作系统内核**强行夺回 CPU 使用权的唯一途径**。
> 2. **变态**：“中断”会使 CPU 从**用户态变为内核态**。
> 3. **并发的基石**：如果没有中断机制，一旦应用程序上 CPU 运行，就会一直运行下去，根本不可能实现“并发”。

## 1. <span style="background:rgba(236, 57, 57, 0.55)">中断的分类</span> (🌟 极高频考点)
在多数教材和考卷中，“中断”特指外中断，而内中断一般统称为“异常”。

### A.<span style="background:rgba(240, 107, 5, 0.2)"> 内中断</span> (也称“<span style="background:#fff88f">异常</span>”、“<span style="background:#fff88f">例外</span>” Exception)
* **本质特征**：与当前执行的指令**有关**，中断信号来源于 <span style="background:rgba(240, 167, 216, 0.55)"><font color="#d83931">CPU 内部</font></span>。
* **具体分类**：
    1. **陷阱/陷入 (Trap)**：由应用程序**故意**引发的。
        * *例子*：应用程序执行“陷入指令”主动请求操作系统内核的服务（即<span style="background:rgba(136, 49, 204, 0.2)">系统调用</span>）。>⚠️ **注意：“陷入指令”本身是一条非特权指令。**
    2. **故障 (Fault)**：由错误条件引起，但**可能被内核程序修复**。
        * *例子*：<span style="background:rgba(136, 49, 204, 0.2)">缺页故障</span>。内核修复后会把 CPU 使用权还给应用程序，让它<span style="background:rgba(240, 107, 5, 0.2)">继续执行</span>。
    3. **终止 (Abort)**：由**致命错误**引起，内核程序无法修复。
        * *例子*：<span style="background:rgba(136, 49, 204, 0.2)">整数除以 0</span>、<span style="background:rgba(136, 49, 204, 0.2)">用户态下企图执行特权指令</span>。此时不再将 CPU 还给应用程序，而是直接强行终止该程序。

### B. <span style="background:rgba(240, 107, 5, 0.2)">外中断</span> (狭义的“中断” Interrupt)
* **本质特征**：与当前执行的指令**无关**，中断信号来源于 CPU **外部**。
* **具体分类**：
    1. **时钟中断**：由<span style="background:rgba(136, 49, 204, 0.2)">时钟部件</span>发来的信号。用于实现时间片轮转，时间片一到就强行剥夺当前程序的 CPU。
    2. **I/O 中断请求**：由<font color="#d83931">输入/输出设备</font>发来的信号。例如<span style="background:rgba(136, 49, 204, 0.2)">打印机打印完成、键盘被按下时，向 CPU 发送信号</span>。

## 2. 中断机制的基本实现原理

中断的处理是一个软硬件结合的过程：

1. **检查中断信号**：
    * **内中断**：CPU 在**执行指令时**会检查是否有异常发生。
    * **外中断**：在**每个指令周期末尾**，CPU 都会检查是否有外中断信号需要处理。
2. **找到对应的中断处理程序 (查表)**：
    * 不同的中断信号，需要用不同的中断处理程序来处理。
    * CPU 检测到中断信号后，会根据信号类型去查询<font color="#d83931"><span style="background:rgba(236, 57, 57, 0.55)">“中断向量表”</span></font>**。
    * **中断向量表**的作用：找到相应的中断处理程序在内存中的存放位置（指针）。     
    *⚠️ **注意**：中断处理程序一定是**内核程序**，需要运行在**内核态**。

## 🎯 3. 知识回顾与硬核映射关系

* **内中断 vs 外中断 判断秘籍**：
    * 问自己一个问题：*“如果当前这条指令不执行，这个中断还会发生吗？”*
    * 如果**不会**发生 👉 是当前指令惹的祸 👉 **内中断（异常）**。（如：除以0，非法指令）
    * 如果**依然会**发生 👉 别人强加进来的 👉 **外中断**。（如：时间片到了，敲击了键盘）
