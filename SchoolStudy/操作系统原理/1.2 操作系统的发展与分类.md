# 操作系统的发展与分类 (1.3)

>[!TIP] **💡 核心学习思路**
> 这一节的核心是理清一条演进主线：**手工操作阶段 ➡️ 批处理阶段 (单道 ➡️ 多道) ➡️ 分时操作系统 ➡️ 实时操作系统**。
> 每一种新系统的诞生，都是为了解决上一代系统的核心缺点。重点记忆各个阶段的**优缺点**及**标志性事件**。

##  1. 手工操作阶段
用户在计算机上进行手工装/取纸带的操作。
* **特点**：用户独占全机。
* **主要缺点（痛点）**：**人机速度矛盾**。手工装/取纸带的速度极慢，而 CPU 计算速度极快，导致 CPU 绝大多数时间都在空闲等待，**资源利用率极低**。

## 2. 批处理阶段

为了解决“人机速度矛盾”，引入了脱机输入/输出技术。
### 2.1 单道批处理系统
引入了**监督程序**（操作系统的雏形）来负责控制作业的输入和输出。
* **主要优点**：缓解了一定程度的人机速度矛盾，资源利用率有所提升。
* **主要缺点**：内存中**仅能有一道程序运行**。只有该程序运行结束后，才能调入下一道程序。CPU 依然有大量时间是在空闲等待 I/O 完成，资源利用率依然很低。

### 2.2 多道批处理系统 (🌟 重点)
每次往内存中读入多道程序。<span style="background:rgba(236, 57, 57, 0.55)">注意：操作系统正式诞生，用于支持多道程序并发运行！</span>
* **主要优点**：多道程序**并发**执行，**共享**计算机资源。**资源利用率大幅提升**，CPU 和其他资源更能保持“忙碌”状态，系统吞吐量增大。
* **主要缺点（痛点）**：用户响应时间长，**没有人机交互功能**。用户提交作业后只能死等结果，中途无法控制、无法调试、无法动态输入参数。

## 3. 分时操作系统
计算机以**时间片**为单位**轮流**为各个用户/作业服务，各个用户可通过终端与计算机进行交互。
* **主要优点**：用户请求可以被即时响应，**解决了人机交互问题**。允许多个用户同时使用一台计算机，且彼此感受不到别人的存在（独立性）。
* **主要缺点（痛点）**：**不能优先处理一些紧急任务**。操作系统对各个用户/作业都是完全公平的，严格循环分配时间片，不区分任务的紧急程度。

## 4. 实时操作系统

为了应对必须紧急处理的任务而诞生。计算机接收到外部信号后及时进行处理，并且要**在严格的时限内处理完事件**。
* **主要特点**：**及时性**和**可靠性**。
* **分类**：
    1. **硬实时系统**：必须在<span style="background:rgba(240, 107, 5, 0.2)">绝对严格的规定时间内完成处理</span>（如：导弹控制系统、自动驾驶系统）。
    2. **软实时系统**：能接受偶尔<span style="background:rgba(240, 107, 5, 0.2)">违反时间规定</span>（如：12306 火车订票系统）。

## 🌐 5. 其他几种操作系统

* **网络操作系统**：实现网络中各种资源的共享（如文件共享）和各台计算机之间的通信（如 Windows NT）。
* **分布式操作系统**：主要特点是**分布性**和**并行性**。系统中各台计算机地位相同，**任何工作都可以分布在这些计算机上**，由它们并行、协同完成。
* **个人计算机操作系统**：如 Windows XP、macOS 等，方便个人使用。

## 🎯 6. 知识回顾与重要考点总结

| 发展阶段      | 解决的核心问题 (优点)  | 存在的主要缺点          | 备注/考点           |
| :-------- | :------------ | :--------------- | :-------------- |
| **手工操作**  | -             | **人机速度矛盾**，利用率极低 | 纯手工，独占全机        |
| **单道批处理** | 缓解人机速度矛盾      | 内存仅一道程序，利用率仍低    | 引入脱机 I/O 和监督程序  |
| **多道批处理** | 并发执行，**利用率高** | **无人机交互功能**      | **操作系统正式诞生**    |
| **分时系统**  | 提供**人机交互功能**  | **不能优先处理紧急任务**   | 以**时间片**为单位轮流服务 |
| **实时系统**  | 能**优先处理紧急任务** | -                | 分为硬实时和软实时       |
