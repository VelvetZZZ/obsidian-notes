# 系统调用 (1.3.3)

> **💡 核心定义**
> 操作系统作为用户和计算机硬件之间的接口，向上提供命令接口和程序接口。
> **程序接口**由一组**系统调用**组成。
> **“系统调用”**是操作系统提供给应用程序（程序员）使用的接口，可以理解为一种可供应用程序调用的特殊函数。应用程序可以通过系统调用来**请求获得操作系统内核的服务**。

## 🆚 1. 系统调用与库函数的区别

在写代码时，我们经常使用编程语言提供的库函数（如 C 语言的 `printf`），它们和系统调用是什么关系？

* **包含关系**：有的库函数是对系统调用的进一步封装（隐藏了底层细节，方便程序员使用）；有的库函数并没有使用系统调用。
* **举例说明**：
    * **不涉及系统调用**的库函数：如“取绝对值” `abs()`，这种纯数学计算不需要操作系统介入。
    * **涉及系统调用**的库函数：如“创建一个新文件”，需要操作硬盘，必须请求操作系统内核服务。

## 🛡️ 2. 为什么必须有系统调用？(什么功能需要用到它？)

**生活场景**：如果两个同学同时用 WPS 和 Word 连着同一台打印机打印论文，如果没有操作系统的统一管理，两篇论文的内容就会混杂在一张纸上。

* **核心根本**：系统中的各种**共享资源**都由操作系统内核统一掌管。凡是与共享资源有关的操作（会直接影响到其他进程），就**必须**通过系统调用的方式向内核提出请求，由内核代为完成。这样可以保证系统的**稳定性和安全性**。

**具体功能分类（这五类必定用到系统调用）**：
1. **设备管理**：设备的请求/释放/启动
2. **文件管理**：文件的读/写/创建/删除
3. **进程控制**：进程的创建/撤销/阻塞/唤醒
4. **进程通信**：进程间的消息传递/信号传递
5. **内存管理**：内存的分配/回收

## 🔄 3. 系统调用的执行过程 (🌟 绝对高频考点！)

系统调用的过程，本质上就是一个**“传参 ➡️ 触发中断（陷入） ➡️ OS 处理 ➡️ 返回”**的过程。

1. **传递参数**：应用程序先将系统调用所需的参数放到某些通用寄存器中。
2. **执行陷入指令 (Trap)**：
    * 应用程序在**用户态**下执行一条**陷入指令**（也称 `trap` 指令、**访管指令**）。
    * **注意**：陷入指令是一条**非特权指令**！
3. **引发内中断，态转换**：
    * 执行陷入指令后，会立即引发一个**内中断**。
    * 硬件自动将 CPU 从**用户态切换为核心态（内核态）**。
4. **操作系统处理**：
    * 操作系统内核程序根据寄存器中的参数，判断用户需要哪种系统调用服务，并在**核心态**下执行相应的处理程序。
5. **返回**：处理完成后，操作系统将 CPU 控制权交还给应用程序，退回用户态。

> ⚠️ **考点避坑指南（默念三遍）**：
> 1. **发出**系统调用请求（执行陷入指令）是在**用户态**！
> 2. **处理**系统调用请求是在**核心态**！
> 3. 陷入指令 = trap指令 = 访管指令，它是**非特权指令**，但它能触发变态！

## 🎯 4. 知识回顾与总结

* **什么是系统调用**：OS 对应用程序/程序员提供的接口。
* **与库函数的区别**：库函数可能封装了系统调用，也可能没有。
* **什么时候用**：凡是涉及**共享资源**管理的操作。
* **核心过程**：传参 👉 陷入指令 (用户态) 👉 引发内中断变态 👉 OS 内核处理 (核心态) 👉 返回。
