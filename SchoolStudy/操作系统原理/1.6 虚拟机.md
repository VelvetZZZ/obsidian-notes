# 虚拟机 (Virtual Machine) (1.6)

> [!info] **💡 核心概念**
> **虚拟机**：使用虚拟化技术，将一台物理机器虚拟化为多台虚拟机器（VM），每个虚拟机器都可以独立运行一个操作系统。
> **同义术语（经常在题目中交替出现）**：虚拟机管理程序 / 虚拟机监控程序 / Virtual Machine Monitor (VMM) / Hypervisor。
> 
## 1. 两类虚拟机管理程序 (VMM) 的对比 (🌟 <font color="#ff0000">核心考点</font>)

根据 VMM 是直接跑在硬件上，还是跑在宿主操作系统上，虚拟机分为两类。这是考试最爱出的对比题：

| 对比维度         | 第一类 VMM (裸金属型)        | 第二类 VMM (宿主型)                                   |
| :----------- | :-------------------- | :---------------------------------------------- |
| **安装位置**     | **直接运行在物理硬件之上**       | 运行在 **==Host OS (宿主操作系统)==** 之上                 |
| **物理资源控制权**  | **直接控制**和分配物理资源       | 依赖 Host OS 为其分配物理资源                             |
| **资源分配方式**   | 分配未经抽象的物理硬件（类似“外核”）   | Guest OS 拥有“虚拟磁盘”（其实是 Host OS 里的一个大文件），分配的是虚拟内存 |
| **🌟 性能**    | **性能更好**              | 性能较差（需要 Host OS 作为中介）                           |
| **支持的虚拟机数量** | 更多（无需和 Host OS 竞争资源）  | 更少（Host OS 本身和其他进程也要吃资源）                        |
| **虚拟机可迁移性**  | 较差                    | **更好**（只需导出镜像文件，如 `.vmdk`，即可在另一台电脑上跑，商业化应用更广泛）  |
| **典型代表**     | VMware ESXi (企业级服务器用) | **VMware Workstation, VirtualBox** (学生/个人常用)    |

##  2. CPU 指令等级与虚拟机的运行模式

支持虚拟化的 CPU 通常会划分更多的指令权限等级，一般分为 Ring 0 ~ Ring 3：
* **Ring 0**：最高权限，可以执行所有指令（特权指令）。
* **Ring 3**：最低权限，只能执行非特权指令。

### 虚拟机环境下的权限分配：
* **第一类 VMM**：
    * **VMM** 运行在最高特权级 (**Ring 0**)。
    * **Guest OS（客户操作系统）** 运行在 **Ring 1**（非最高权限，但有部分特权）。
    * 用户应用程序 运行在 **Ring 3**。
    * *机制*：当 Guest OS 企图执行**敏感指令**时，会被 Ring 0 的 VMM 拦截并代为处理。
* **第二类 VMM**：
    * VMM 部分运行在用户态，部分运行在内核态。
    * Guest OS 发出的系统调用会被 VMM 截获，并转化为 VMM 对 Host OS 的系统调用。

##  3. 知识回顾与排雷提示

* **理清概念**：
    * **Host OS**：你的物理电脑上原本装的系统（比如你买来电脑自带的 Windows 11）。
    * **Guest OS**：你在虚拟机软件里装的系统（比如在 VMware 里装的 Ubuntu Linux）。
* **考试雷区**：记住 **“第一类性能好，第二类好迁移”**。我们在个人电脑上装的都是第二类，不要因为觉得 VMware 很厉害就误以为它是第一类。
