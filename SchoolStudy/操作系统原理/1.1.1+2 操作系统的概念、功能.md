# 操作系统的概念、功能与目标

> [!IMPORTANT] **💡 核心定义与角色**
> **操作系统（Operating System, OS）**是指控制和管理整个计算机系统的硬件和软件资源，并合理地组织调度计算机的工作和资源的分配；以提供给用户和其他软件方便的接口和环境；它是计算机系统中最基本的**系统软件**。
>
> **核心提炼（OS的三个角色）：**
> 1. 是系统资源的**管理者**
> 2. 向上层提供**方便易用的服务**
> 3. 是**最接近硬件**的一层软件

##  1. 作为系统资源的管理者

操作系统负责协调硬件、软件等计算机资源的工作。目标是做到**安全、高效**。

主要包括以下四个方面的管理（以QQ视频聊天为例）：
* ==**文件管理**==：找到安装目录下的 `QQ.exe` 可执行文件。
* ==**存储器（内存）管理**==：双击运行后，需要把该程序的相关数据放入内存。
    *  **补充考点：**<span style="background:rgba(240, 107, 5, 0.2)"> 执行一个程序前**必须**将该程序放到内存中，才能被CPU处理。</span>
* ==**处理机（CPU）管理**==：QQ程序对应的进程被分配到处理机（CPU）上运行。
* ==**设备管理**==：将摄像头、麦克风等硬件设备分配给该进程使用。

##  2. 向上层提供方便易用的服务

**封装思想**：操作系统把一些丑陋的、复杂的纯硬件指令（如二进制代码 010101...）封装成简单易用的服务，暴露给用户“美丽”的接口。用户无需关心底层硬件原理，只需对OS发出命令即可。

系统提供的<span style="background:#fdbfff">接口</span>主要分为两类：

### 1. 给普通用户使用的（<span style="background:#fdbfff">用户接口</span>）
* **GUI（图形用户界面）**：如 Windows、macOS、iOS 的图形化操作界面，所见即所得。
* **命令接口**：
    * **联机命令接口（交互式）**：特点是<span style="background:rgba(240, 167, 216, 0.55)">“用户说一句，系统跟着做一句”</span>。例如 Windows 的 `cmd` 命令行，输入 `time` 命令，系统立刻返回时间。
    * **脱机命令接口（批处理）**：特点是<span style="background:rgba(240, 167, 216, 0.55)">“用户说一堆，系统跟着做一堆”</span>。例如 Windows 中的 `.bat` 批处理文件，里面写满指令，运行后系统会一次性批量执行。

>[!WARNING] 狭义的用户接口不包含GUI
### 2. 给软件/程序员使用的（<span style="background:#fdbfff">程序接口</span>）
* **程序接口（即“系统调用”）**：普通用户不能直接使用，只能通过编写程序代码**间接**使用。
    * 例如：写 C 语言的 `printf` 函数时，底层其实就是调用了操作系统提供的“显示相关”的系统调用。
    * **重要考点：** <span style="background:rgba(240, 107, 5, 0.2)">系统调用类似于函数调用，是应用程序请求操作系统服务的</span>==**唯一方式**==。
    * **名词替换：** 在有的教材中，==**系统调用 = 广义指令**。==

## 3. 作为最接近硬件的层次（对硬件机器的拓展）

* 没有没有任何软件支持的计算机称为**裸机**。
* 在裸机上安装操作系统，可以提供资源管理功能和方便用户的服务，将其改造成为功能更强、使用更方便的机器。
* **扩充机器（虚拟机）**：通常把覆盖了软件的机器称为扩充机器。
	* 操作系统对硬件机器的拓展：将CPU、内存、磁盘、显示器、键盘等硬件合理地组织起来，让各种硬件能够相互协调配合，实现更多更复杂的功能（类似汽车发动机加传动系统）。

##  4. 知识回顾与重要考点总结

* **操作系统的概念**
    * 负责管理协调硬件、软件等资源
    * 为上层提供简单易用的服务
    * 是一种系统软件
* **功能和目标**
    * **资源的管理者**：处理机管理、存储器管理、文件管理、设备管理
    * **向上层提供服务**：
        * 给普通用户：GUI、命令接口（联机/脱机）
        * 给程序员：程序接口（即**系统调用**）
    * **对硬件机器的拓展**：将裸机改造为扩充机器（虚拟机）
