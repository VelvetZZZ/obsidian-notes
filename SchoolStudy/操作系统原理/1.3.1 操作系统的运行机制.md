# 操作系统的运行机制 (1.3.1)

> [!TIP] **💡 核心逻辑纵览 (三个“两”)**
> 操作系统的运行机制可以浓缩为三个二元对立的概念：
> 1. **两类程序**：内核程序 vs 应用程序
> 2. **两类指令**：特权指令 vs 非特权指令
> 3. **两种处理器状态**：内核态 vs 用户态

##  0. 预备知识：程序是如何运行的？

* 程序员用 C 语言等高级语言编写的代码，经过编译器的“翻译”，最终变成 CPU 能识别的**机器指令（二进制）**。
* 程序运行的过程，其实就是 CPU 逐条<span style="background:rgba(240, 107, 5, 0.2)">执行机器指令</span>的过程。
* *注意区分：这里的“指令”特指==二进制机器指令==，不是指你在终端敲的 `ls` 或 `cd` 这种交互式命令。*

## 💻 1. 两类程序与两类指令

### 两类程序

* **应用程序**：普通程序员写的程序（如 QQ、微信）。
* **内核程序**：微软、苹果等大厂工程师写的，负责实现操作系统的核心功能。很多个内核程序组成了“**操作系统内核**”，简称“**内核 (Kernel)**”。
    * ⚠️ **易错考点**：内核是操作系统最重要、最核心的部分。但**操作系统的功能未必都在内核中**（例如：图形化用户界面 GUI 就不在内核中）。

### 两类指令
CPU 在设计生产时就划分了指令的权限级别：
* **非特权指令**：破坏性小、普通的指令（如加法、减法）。**应用程序**只能使用非特权指令。
* **特权指令**：影响重大、有破坏性的指令（如内存清零指令）。只允许“管理者”——即**内核程序**使用。

## 2. 两种处理器状态 (CPU 状态)
既然有特权指令，CPU 在执行前必须知道当前是谁在发号施令。因此 CPU 内部被划分了两种状态：

* **用户态（目态）**：说明此时正在运行的是**应用程序**，此时 CPU 只能执行**非特权指令**。
* **内核态（核心态、管态）**：说明此时正在运行的是**内核程序**，此时 CPU 可以执行**特权指令**。

> [!IMPORTANT] 🔑 **底层机制拓展**：
> CPU 内部有一个**程序状态字寄存器 (PSW)**。PSW 中有一个特定的二进制位代表状态（例如：1 代表内核态，0 代表用户态）。

## 🔄 3. 内核态与用户态的切换 (🌟 极高频考点)
操作系统在运行过程中，CPU 会在两种状态之间反复横跳（俗称“变态”）：

### ➡️ 内核态 切换到 用户态
* **触发方式**：操作系统主动让出 CPU 控制权。
* **具体操作**：内核程序执行一条**特权指令**，修改 PSW 的标志位为“用户态”。

### ⬅️ 用户态 切换到 内核态 (核心考点！)
* **触发方式**：由**“中断” (Interrupt)** 引发，这是一个由**硬件自动完成**的过程。
* **具体含义**：触发中断信号意味着操作系统将**强行夺回** CPU 的使用权。
* **触发场景**：只要是需要操作系统介入的地方，都会触发中断（例如：应用程序企图非法执行特权指令，此时硬件会立刻触发中断，强行将 CPU 切回内核态，由 OS 来处理这个非法行为）。

## 🎯 4. 知识回顾与硬核映射关系

一定要把这几者的对应关系牢牢绑定死：

* **内核程序** 运行在 👉 **内核态 (管态/核心态)** 👉 可以执行 👉 **特权指令 + 非特权指令**
* **应用程序** 运行在 👉 **用户态 (目态)** 👉 只能执行 👉 **非特权指令**

* **变态口诀**：
    * 内核 变 用户：**特权指令**（OS 主动下放权力）
    * 用户 变 内核：**中断**（硬件强行介入交还 OS）
