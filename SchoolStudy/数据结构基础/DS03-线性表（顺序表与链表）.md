# 判断题
### 1-1 If the most commonly used operations are to <span style="background:#d2cbff">visit a random position</span> and to <span style="background:rgba(240, 167, 216, 0.55)">insert and delete the last element in a linear list</span>, then <span style="background:rgba(236, 57, 57, 0.55)">sequential storage </span>works the fastest. <font color="#ff0000">T</font>

翻译：
如果在一个**线性表 (linear list)** 中，最常用的操作是 <span style="background:#d2cbff">访问一个随机位置</span> (visit a random position) 和 <span style="background:rgba(240, 167, 216, 0.55)">在最后一个元素处插入和删除</span> (insert and delete the last element)，那么<span style="background:rgba(236, 57, 57, 0.55)">**顺序存储 (sequential storage)**</span> 工作得最快。

##### “访问一个随机位置” (如 `arr[i]`)
- ###### <span style="background:#fdbfff">"顺序存储" (数组)</span> - 速度：O(1) (瞬时)
    
    - **(它的“超能力”)** 这是“数组”**最强**的地方。
        
    - 内存是“连续”的。C语言可以**“瞬间”计算**出 `arr[i]` 的地址 (`基地址 + i * 元素大小`)。
        
    - **评价：** 完美。
        
- ###### <span style="background:#fdbfff"> "链式存储" (链表)</span> - 速度：O(n) (龟速)
    
    - **(它的“致命弱点”)** 这是“链表”**最弱**的地方。
        
    - 内存是“分散”的。你**没有**“捷径”！
        
    - 你**必须**从 `head` (车头) 出发，一步一步（`->next`）“走” `n` 步，才能找到第 `n` 个元素。
        
    - **评价：** 灾难。

##### “在最后一个元素处插入和删除” (Push/Pop at End)

- ###### "顺序存储" (数组) - 速度：O(1) (瞬时)
    
    - 我们在实现“顺序表”时，会有一个变量 `count` 记录“当前有多少个元素”。
        
    - **插入 (Insert at end)：** `arr[count] = new_data; count++;`
        
    - **删除 (Delete at end)：** `count--;` (我们甚至都不用擦除数据，直接把 `count` 减 1 就行了)
        
    - **评价：** 极其高效，**瞬时**完成。
        
- ###### "链式存储" (链表) - 速度：O(1) 或 O(n)**
    
    - **插入 (Insert at end)：** 如果你有“尾指针” (`tail`)，`tail->next = ...`，这是 `O(1)`。
        
    - **删除 (Delete at end)：** **(陷阱！)**
        
        - 如果是“**单**向链表”，你**必须**从 `head` 出发，“走” `n` 步，找到“**倒数第二个**”元素，才能删除“最后一个”。这是 **O(n)**。
            
        - 如果是“**双**向链表”，你可以用 `tail->prev` 找到“倒数第二个”，这是 `O(1)`。
            
    - **评价：** 速度**不稳定**，且**更复杂**。

# 选择题

### 2-1 For a sequentially stored linear list of length N, which of the following operations has the time complexity O(1)?

A. visit the i-th (1≤i≤N) node and find the immediate predecessor of the i-th (2≤i≤N) node

B. insert a new node after the i-th (1≤i≤N) node

C. delete the i-th (1≤i≤N) node

D. sort the N nodes in increasing order

- 选项 A 中的“访问第 $i$ 个节点”（即 `arr[i]`）和“找到第 $i$ 个节点的直接前驱”（即 `arr[i-1]`）**都是**随机访问操作，它们的时间复杂度都是 **$O(1)$**。
- 选项 B 和 C（插入和删除）需要移动大量元素，是 **$O(n)$**。
- 选项 D（排序）至少是 **$O(n \log n)$**。
