#### <span style="background:rgba(236, 57, 57, 0.55)">递归 (Recursion)</span>：
##### 1. 定义：
在计算机科学中，递归是一种**解决问题的方法**，它将一个**复杂问题**分解为一个或多个**结构相同**但**规模更小**的**子问题**，直到子问题变得足够简单，可以直接求解。

一个**递归函数 (Recursive Function)** 是指在其定义中<span style="background:#d2cbff">直接或间接调用其自身的函数</span>。

##### 2. 必要组成 

一个**正确**的递归函数**必须**包含两个核心组成部分，缺一不可：

1. **基本情况 (Base Case / Terminating Condition)**
    
    - **定义：** 问题的“最小版本”，它**不需要**进一步递归**就可以直接求解**。
        
    - **作用：** 这是递归的**“停止信号”**或**“出口”**。
        
    - **示例 (以“阶乘”为例)：** 0的阶乘 ( `0!` ) 是 1。这是一个已知的、无须再计算的事实。
        
2. **递归步骤 (Recursive Step / Reduction Step)**
    
    - **定义：** 将当前问题分解为一个或多个“更小”的、与原问题“同类”的子问题。
        
    - **作用：** **调用函数自身**来解决这些子问题，并**组合**子问题的解来构成原问题的解。
        
    - **关键：** 递归调用所使用的参数**必须**以某种方式**“更接近”**基本情况（这个过程称为**“收敛”**）。
        
    - **示例 (以“阶乘”为例)：** 5的阶乘 ( `5!` ) 是 `5 * (4!)`。它把 `factorial(5)` 问题，归约 (reduce) 成了求解 `factorial(4)` 这个“更小”的“同类”问题。

##### 3.特点：
- **简洁易懂” (优点)：**
    
    - 递归就是“函数调用自己”。
        
    - 对于某些问题（比如“求阶乘”、“遍历树”），它的逻辑和数学定义完全一样，写出来的代码非常“优雅”。
        
- **“占用较大空间” (缺点)：**
    
    - **比喻：** 想象“俄罗斯套娃”。
        
    - 你调用函数（打开第1层套娃），它为了解决问题，又调用了自己（打开第2层套娃），然后又调用了自己（打开第3层）...
        
    - **关键：** 每当你“打开一个新套娃”时，你**必须记住**你“外层的套娃”长什么样，以便等下能把它“关回去”。
        
    - 这个“记忆”所占用的空间，就叫做 **“栈” (Stack)**。
        
- **“造成系统堆栈溢出” (Stack Overflow)：**
    - 如果你的“递归层次过大”（比如你一次要开100万层套娃）。
        
    - 你的“记忆”空间（栈）是**有限的**。
        
    - 当你把“栈”塞满了，再想“打开一层”时，系统就会崩溃，这就是<span style="background:rgba(236, 57, 57, 0.55)">“栈溢出” </span>(Stack Overflow)。
### 1-2 在函数内部出现的变量与全局变量同名，若未对该变量赋值，则为全局变量。<font color="#ff0000">T</font>

##### 核心概念：<span style="background:#d2cbff">“遮蔽” (Shadowing)</span>
当你使用一个变量 `x` 时，C语言会<span style="background:rgba(236, 57, 57, 0.55)">“由内向外”</span>找它：
1. **先看“本地” (函数内)：** “我这个函数里，有没有**声明** (declare) 一个叫 `x` 的变量？” (比如 `int x;` 或 `float x;`)
    
    - **如果找到了：** OK，**停止寻找**。在这个函数里，所有的 `x` **都**是这个“本地 `x`”。这个本地 `x` **“遮蔽” (Shadows)** 了外面的 `x`。
        
    - **如果没找到：** C语言会“往外看一层”。
        
2. **再看“全局” (函数外)：** “函数外面，有没有**声明**一个全局的 `x`？”
    
    - **如果找到了：** OK，我就用这个“全局 `x`”。
        
    - **如果没找到：** 编译器报错：“`x` 未定义”。

### 1-3 指向<span style="background:rgba(240, 167, 216, 0.55)">结构体变量</span>的指针可以作<span style="background:rgba(240, 167, 216, 0.55)">函数参数</span>，实现<font color="#ff0000">传址调用</font>。<font color="#ff0000"> T</font>

##### <span style="background:rgba(236, 57, 57, 0.55)">传址调用的定义</span>

> **原题：** “指向结构体变量的指针 (1) 可以作函数参数 (2)，实现传址调用 (3)。”
> 
> **翻译：** “你手里的那张‘便条’(1)(上面写着‘档案袋’的存放地址)，可以被递交给‘办事处’的‘收件窗口’(2)。这个<span style="background:rgba(240, 107, 5, 0.2)">‘递便条’</span>的动作 (3)，就叫‘传址调用’。”

在数据结构中，当你想让一个函数**真正地修改**你的链表（比如“插入”或“删除”节点）时，你**必须**使用这种“传址”的方式。

##### C语言两种**完全不同**的“办事”方式：传址 vs. 传值
###### 方式一 (传值 / Pass-by-Value)：低效且“自私”

- **动作：** 你把你的**100页“档案袋” (结构体)**，拿到复印机上**完整地复印 (Copy) 了一份**，然后把这份**“复印件”**交给了“收件窗口”。
    
- **缺点1 (慢)：** 如果“档案袋”有100MB，复印就要花很久时间。
    
- **缺点2 (无效)：** “办事处”在“复印件”上疯狂修改、盖章。**你的“原件”一动没动**，修改无效！
    

###### 方式二 (传址 / Pass-by-Address)：高效且“共享” (本题)

- **动作：** 你**不复印**任何东西。你只把那张写着“档案室柜子号”的**“便条” (指针)** 交给了“收件窗口”。
    
- **优点1 (快)：** 递一张“便条”是**瞬间**完成的 (O(1))。
    
- **优点2 (有效)：** “办事处”拿到了“地址”，**直接跑到档案室**，找到了你的**“原件”**！**所有的修改都发生在“原件”上！**

##### 1. “作用域”的陷阱

核心：**这里有两对 `x` 和 `y`！**

- **第一对：全局 (Global) `x` 和 `y`**
    - 在**所有函数外面** `int x = 5, y = 7;`
    - 它们就像“**大厅**”里的两个公告牌。

- **第二对：`main` 的本地 (Local) `x` 和 `y`**
    - 在 `main` 函数**内部** `int x = 3, y = 8;`
    - 它们就像 `main` **办公室自己桌上**的“便条”。
