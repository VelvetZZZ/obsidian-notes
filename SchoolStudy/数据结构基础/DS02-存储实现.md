##### 考点：C++ 的 `iostream` 库（即 `cout`）的特性，这个特性叫做<span style="background:rgba(240, 167, 216, 0.55)">“运算符重载”</span> (Operator Overloading)
1. <font color="#b2a2c7">“可以输出各种基本数据类型”：</font>
- 在 C++ 中，`<<` 操作符被“重载”了。这意味着 C++ 标准库为**每一种**基本类型（`int`, `double`, `float`, `char`, `bool` 等）都提供了一个 `<<` 的版本。
e.g.
- 当你写 `cout << 5;` 时，它调用的是 `int` 版本的 `<<`。
    
- 当你写 `cout << 3.14;` 时，它调用的是 `double` 版本的 `<<`
1. <font color="#b2a2c7">“也可以输出指针值”：</font>
- C++ 标准库**同样**为**指针**（`void*` 类型）重载了 `<<`。
e.g.
- 当你写 `cout << ptr;` (无论 `ptr` 是 `int*` 还是 `char*`)，它会调用指针版本的 `<<`。
    
- 此时，它输出的**不是**指针“指向”的值，而是**指针“本身”的值**——即它所存储的那个**内存地址**（通常是一长串十六进制数，比如 `0x7ffc1234abc`）。

### 1-3 使用new运算符开辟存储空间后，不必使用delete运算符撤销相应的空间。<font color="#d83931">F</font>
##### <font color="#8064a2">C++ 内存管理 (new/delete)</font>
###### 1. Python (你的“母语”)：有“自动管家”
- 在 Python 中，当你写 `my_list = [1, 2, 3]` 时，Python 解释器会自动帮你“申请一块内存”来存这个列表。

- 当你不再使用 `my_list` 时（比如程序结束了，或者你 `del my_list`），Python 的**“垃圾回收器” (Garbage Collector, GC)** 会自动发现“这块内存没人用了”，然后**自动**帮你**“释放”**它。

- 你（程序员）**完全不用操心**内存的“申请”和“释放”。
###### 2. C++ / C (考试语言)：没有“自动管家”

- 在 C++ 中，Python 的“垃圾回收器”**根本不存在**！

- C++ 把内存管理的**全部责任**都交给了你（程序员）。

- **`new` 是什么？**
    
    - `new` 是你**手动**向操作系统<span style="background:rgba(136, 49, 204, 0.2)">“申请内存”</span>的命令。
    - 比如 `int* ptr = new int;`
    
    - **人话：** “操作系统，请在**堆 (Heap)** 上给我一块能存 `int` 的内存，我会用 `ptr` 这个指针记住它的地址。”
    
- **`delete` 是什么？**
    
    - `delete` 是你**手动**向操作系统<span style="background:rgba(136, 49, 204, 0.2)">“归还内存”</span>的命令。
    - 比如 `delete ptr;`
    
    - **人话：** “操作系统，我用完 `ptr` 指向的那块内存了，现在还给你。”
###### 3. 结论 (为什么题目是错的)

- **题目说：** “`new` 申请了，**不必** `delete` 归还。”
- **这是 100% 错误的！**
- **如果你只 `new` 而不 `delete`，会发生什么？**
    
    - 你申请了一块内存，用完了，但你**没有归还**它。
        
    - 这块内存就“丢失”了——操作系统以为你还在用它，所以不会把它交给别人；而你已经“忘了”它的地址，你也无法再使用它。
        
    - 这个bug叫做<span style="background:rgba(240, 167, 216, 0.55)"> “内存泄漏” (Memory Leak)</span>。
        
    - 如果你的程序在一个循环里不停地 `new`，而不 `delete`，你的程序占用的内存会无限增长，直到耗尽所有系统资源，最终崩溃。

### 1-5 当函数的返回值是<span style="background:rgba(136, 49, 204, 0.2)">数组类型</span>的，传递的是数组<span style="background:#fdbfff">第一个元素</span>的地址。<font color="#d83931"> T</font>

##### 在C语言中，**<span style="background:#fdbfff">“数组名” 就等于 “数组第一个元素的地址”</span>**。

>所以，无论你是把数组作为参数传递给函数，还是作为返回值返回，你操作的**都不是整个数组的拷贝**，而仅仅是那个<font color="#953734">指向开头的地址</font>。
##### 知识点一：数组在内存中是什么样的？

想象一下，一个数组 `int arr[10]` 就像是在内存里预订了**一排10个连续的储物柜**。

- `arr` 这个名字，并不代表“所有10个储物柜里的东西”，它代表的是 **“第一个储物柜的门牌号（地址）”**。
    
- 当你访问 `arr[0]`，你是去第一个储物柜取东西。
    
- 当你访问 `arr[3]`，C语言会帮你计算：从第一个储物柜的地址开始，往后数3个柜子，然后去那个地址取东西。

##### 知识点二：C语言的“效率陷阱”与“指针”

现在，一个函数（我们叫它 `function_A`）创建了这个数组 `arr`，并想把它交给另一个函数 `function_B`。

- **低效的做法（C语言禁止的）：** 把这10个储物柜里的所有东西都复制一遍，再交给 `function_B`。如果这个数组有100万个元素呢？这种拷贝会极其缓慢。
    
- **高效的做法（C语言采用的）：** `function_A` 不拷贝任何东西，它只递给 `function_B` 一张纸条，上面写着 **“第一个储物柜的地址”**。

这张“写着地址的纸条”，在C语言里就叫做 **“指针” (Pointer)**。

##### 结论：回到你的题目

> “当函数的返回值是数组类型的，传递的是数组第一个元素的地址。”

这句话其实在C语言语法上是一个“伪命题”，因为**C语言不允许函数直接返回一个“数组类型”**。你不能写这样的代码：`int[10] my_func() { ... }`。

你**必须**这样写：`int* my_func() { ... }`。 这里的 `int*` 就是在告诉编译器：“我这个函数返回的是一个**指针**（地址），这个地址指向一个整数（通常是数组的第一个元素）。”

所以，这道题的真实含义是： “<span style="background:rgba(240, 167, 216, 0.55)">因为C语言不允许你返回整个数组，所以你唯一能做的是返回一个指针，这个指针指向的就是数组第一个元素的地址。</span>”

因此，该描述为 **True**。

##### **数组 (Array)** 和 **链表 (Linked List)** 区别的基础：

- **数组 (Array)：**
    - **内存：** 像储物柜一样，**连续**排列。
        
    - **访问：** 访问非常快。因为你知道第一个元素的地址（`arr`），你可以通过简单的数学计算（`地址 + 索引 * 元素大小`）**立即**跳到任何一个元素，这叫**随机访问 (Random Access)**。
        
    - **传递：** 传递成本极低，只传递一个地址（指针）。
        
- **链表 (Linked List)：**
    
    - **内存：** 元素（节点）在内存中是**分散**的，东一个西一个。
        
    - **访问：** 访问很慢。你必须从第一个元素开始，顺着它存储的“下一个元素的地址（指针）”，一步一步跳过去，这叫**顺序访问 (Sequential Access)**。
        
    - **传递：** 传递成本也极低，只传递第一个元素的地址（头指针）。

##### `i=j=10;` 这是本题的**核心陷阱**。
1. **这不是比较！** 这是**赋值**。
2. C语言的赋值是从右往左执行的：
    - 首先，执行 `j = 10`。`j` 的值被**重置**为 `10`。
        
    - 然后，执行 `i = (j的值)`。`i` 的值被**重置**为 `10`。

3. **判断真假：** 在C语言中，任何**非零值**都被认为是 `true` (真)，只有 `0` 才被认为是 `false` (假)。
    
    - 整个表达式 `i=j=10` 的最终值是 `10`。
        
    - 因为 `10` 不是 `0`，所以条件**永远为 `true`**。
- **当前状态 (第一次检查后)：** `i=10`, `j=10`。条件为 `true`，**进入循环**。
##### <font color="#ff84ac">C 语言的 “真/假” 判断逻辑</font><font color="#ff84ac">：</font>
C 语言没有 Python 里的 `True` 和 `False` 布尔类型 (在后来的标准里才加入，但底层逻辑不变)。C 语言的逻辑非常“简单粗暴”：
- **`0` (整数0)** 等价于 **`false` (假)**
- **任何非 `0` 的值** (比如 `1`、`10`、`-5`、`3.14`) 等价于 **`true` (真)**
##### <font color="#c00000">C语言中的for循环和python完全不一样！</font>
###### C 语言 `for` 循环的逻辑 (它是一个“伪装”的 `while` 循环)
你把它看成一个“计数器”或“<span style="background:rgba(240, 107, 5, 0.2)">条件循环</span>”更合适。

它的完整结构是： `for ( [1. 初始化] ; [2. 条件判断] ; [3. 循环后更新] )` `{` `// [4. 循环体]` `}`

它的**执行顺序**是这样的：

1. **执行 [1. 初始化]**：这步**只在最开始做一次**。（比如 `int i=0`）
    
2. **执行 [2. 条件判断]**：
    
    - 计算这个表达式的值。（比如 `i < 10`）
        
    - 判断这个值**是不是 `0`**。
        
    - 如果**是 `0` (false)**：立刻**跳出**整个 `for` 循环，结束。
        
    - 如果**不是 `0` (true)**：继续下一步。
        
3. **执行 [4. 循环体]**：运行 `{}` 内部的所有代码。
    
4. **执行 [3. 循环后更新]**：执行更新语句。（比如 `i++`）
    
5. **跳回到第 2 步**，重新进行**条件判断**。
    

**关键点：** C语言的 `for` 循环，本质上就是 `while` 循环的一个“豪华版”（更方便、更清晰、更结构化）。
e.g `while` 循环实现“从0数到9”：
```c
// [1. 初始化]
int i = 0; 

// [2. 条件判断]
while (i < 10) { 
    
    // [4. 循环体]
    printf("%d\n", i); 
    
    // [3. 循环后更新]
    i++; 
}
```
`for` 循环：
```c
// [1. 初始化]; [2. 条件判断]; [3. 循环后更新]
for (int i = 0;   i < 10;          i++) 
{
    // [4. 循环体]
    printf("%d\n", i); 
}
```
###### Python `for` 循环的逻辑 (它是“迭代器”)

Python 的 `for` 循环完全是另一种东西。它叫做 **“for-each” 循环**或<span style="background:rgba(236, 57, 57, 0.55)">“迭代循环”</span>**。

`for item in iterable:`

它的逻辑是：

1. **“请给我一个‘可迭代’的东西”**（比如一个 `list`、一个 `range()`、一个字符串）。
    
2. **“我会自动从里面一个一个地往外取东西”**，每取出一个，就把它赋值给 `item`。
    
3. **“当这个东西被取完时，我就自动停止”**。
    

Python的 `for` 循环<font color="#953734">没有“条件判断”这个概念</font>。它的停止条件是“东西取完了”。

>**记忆口诀：** 在C语言的混合运算中，**“精度”就像一个会传染的“老大”**。只要有“老大”在，所有“小弟”都会被“提升”成和“老大”一样的级别，最终结果也是“老大”的级别。

### 2-4 在一个长度为n（n>1）的<span style="background:rgba(236, 57, 57, 0.55)">单链表</span>上，设有头和尾两个指针，执行（）操作与<span style="background:rgba(136, 49, 204, 0.2)">链表的长度</span>有关。
A.删除单链表中的第一个元素

<span style="background:#fff88f">B.删除单链表中的最后一个元素</span>

C.在单链表第一个元素前插入一个新元素

D.在单链表最后一个元素后插入一个新元素

**记忆口诀** ：单链表是<font color="#c00000">“单行道”</font>，你永远不能“回头”。

##### 解释：<font color="#c00000">B. 删除最后一个元素 (删除车尾)</font>

1. **操作：** 乘务员 `tail` (在车尾，比如第 `n` 节) 想把自己所在的车厢删掉。

2. **步骤 (陷阱来了！)：**
    
    - 要删除 `[车尾]` (第 `n` 节)，我们必须让它**前面那节车厢** (第 `n-1` 节) 的“挂钩”解开。
        
    - 同时，乘务员 `tail` 必须移动到第 `n-1` 节车厢，说：“你现在是新车尾了。”
        
3. **核心问题：** 乘务员 `tail` 在第 `n` 节车厢，**他怎么找到第 `n-1` 节车厢？**
    
4. **答案：** 他**找不到**！因为挂钩是**单向**的，他**不能“回头”**。
    
5. **唯一的解决办法：**
    
    - `tail` 只能用对讲机呼叫**车头的 `head` 乘务员**。
        
    - `head` 乘务员从 `[车头]` (第 1 节) **开始**，**一节一节**地往后走。
        
    - 他走到 `[车厢 2]`，问：“你的下一节是车尾吗？”“不是。”
        
    - 他走到 `[车厢 3]`，问：“你的下一节是车尾吗？”“不是。”
        
    - ...
        
    - 他走到 **第 `n-1` 节车厢**，问：“你的下一节是车尾 (`tail`) 吗？”“**是！**”
        
    - **停！** `head` 乘务员终于找到了这个“倒数第二个”元素。
        
6. **耗时：** <span style="background:#b1ffff">`head` 乘务员必须“遍历” (walk through) 几乎整列火车，这个“走路”的时间，取决于火车的长度 `n`。火车越长，他走得越久。</span>
    
7. **结论：** **O(n) 线性时间**，**与 `n` 强相关**。

##### <span style="background:rgba(236, 57, 57, 0.55)">单链表与双链表</span>
###### 1. 单向链表 (Singly Linked List)
<font color="#d83931">核心比喻：单向的“火车”</font>

- **节点 (Node)：** 每一节**车厢**。
    
- **挂钩 (Pointer)：** 车厢之间**单向的挂钩**。你只能从 1 号车厢走到 2 号，**永远不能**从 2 号走回 1 号。

<font color="#d83931">结构 (C语言中的样子)：</font>

每个“节点”（车厢）包含两样东西：

1. **数据 (data)：** 车厢里装的“货物”。
    
2. **`next` 指针：** 指向**下一节车厢**的“挂钩”。

```c
// 单向链表节点的“设计图”
struct Node {
    int data;            // 货物
    struct Node* next;   // 指向下一个节点的“挂钩”
};
```

 <font color="#d83931">内存中的样子 (图示):</font>
`Head` 和 `Tail` 是两个“乘务员”（指针），他们**不是车厢**，他们只是**指向**车厢。


```Plaintext
[Head]
         |
         v
NULL <-> +------+------+ <-> +------+------+ <-> +------+------+ <-> NULL
       | prev |  10  | next | | prev |  20  | next | | prev |  30  | next |
       +------+------+     +------+------+     +------+------+
                                                       ^
                                                       |
                                                     [Tail]
```

- **`Head`：** 永远指向**第一个**节点。
- **`Tail`：** 永远指向**最后一个**节点。
- **`NULL`：** 最后一节车厢的 `next` 挂钩**什么都不指**，表示“到头了”。

 优点:
- **节省空间：** 每个节点只有一个 `next` 指针，内存占用最小。
- **结构简单：** 实现起来最容易。

缺点:
- **单行道：** **你永远不能回头。** 一旦你走到了 3 号车厢，你就失去了所有关于 1 号和 2 号车厢的信息（除非你从 `Head` 重新走）。

- **这就是上一题的考点：**
    
    - **`删除最后一个元素 (Tail)` 为 O(n)**。
        
    - **原因：** `Tail` 乘务员在 `[30]` 号车厢，他想删掉自己。但他必须告诉**前一节车厢** `[20]` 把挂钩 `next` 设为 `NULL`。
        
    - **他回不去！** 他只能从 `Head` 开始，重新走 `10 -> 20`，才能找到那个“倒数第二”的节点。这个“走路”的时间取决于链表长度 `n`。
        

---

###### 2. 双向链表 (Doubly Linked List)
<font color="#d83931">核心比喻：双向的“地铁”</font>

- **节点 (Node)：** 每一节**车厢**。
    
- **挂钩 (Pointer)：** 车厢之间有**两个挂钩**（`next` 和 `prev`）。你可以从 2 号车厢**走回** 1 号，也可以**走进** 3 号。

 <font color="#d83931">结构 (C语言中的样子):</font>

每个“节点”（车厢）包含三样东西：

1. **数据 (data)：** 货物。
    
2. **`next` 指针：** 指向**下一节车厢**的“挂钩”。
    
<span style="background:rgba(136, 49, 204, 0.2)">3. `prev` 指针：指向上一节车厢的“挂钩”。</span>

```c
// 双向链表节点的“设计图”
struct Node {
    int data;            // 货物
    struct Node* next;   // 指向下一个节点的“挂钩”
    struct Node* prev;   // 指向前一个节点的“挂钩”
};
```

<font color="#d83931">内存中的样子 (图示):</font>

```Plaintext
       [Head]
         |
         v
NULL <-> +------+------+ <-> +------+------+ <-> +------+------+ <-> NULL
       | prev |  10  | next | | prev |  20  | next | | prev |  30  | next |
       +------+------+     +------+------+     +------+------+
                                                       ^
                                                       |
                                                     [Tail]
```

- **`Head` 的 `prev` 指针** 指向 `NULL`。
    
- **`Tail` 的 `next` 指针** 指向 `NULL`。
    
- 中间的每个节点都和前后邻居“紧密相连”。

<font color="#d83931">优点:</font>
- **双向通行：** **极其灵活**。你可以从任何一个节点出发，向前或向后遍历整个链表。
    
- **解决了上一题的痛点：**
    
    - **`删除最后一个元素 (Tail)` 为 O(1) (瞬时)**。
        
    - **原因：** `Tail` 乘务员在 `[30]` 号车厢。
        
    - `Tail` 想删掉自己，他**不需要**从 `Head` 跑过来。
        
    - 他只需要通过自己的 **`prev` 挂钩**，**瞬间**找到 `[20]` 号车厢。
        
    - 然后他对 `[20]` 号车厢说：“把你的 `next` 挂钩解开 (设为 `NULL`)”。
        
    - `Tail` 乘务员再跳到 `[20]` 号车厢，任务完成。这与火车长度 `n` **完全无关**。

<font color="#d83931">缺点:</font>
- **占用更多空间：** 每个节点都需要额外存储一个 `prev` 指针。如果有100万个节点，就多了100万个指针的内存开销。
    
- **实现更复杂：** 在**插入**或**删除**节点时，你必须同时维护 `next` 和 `prev` 两个挂钩。
    
    - `插入`：需要修改 4 个指针。
        
    - `删除`：需要修改 2 个指针。
        
    - **极易出错：** 只要你忘记更新其中任何一个，整个链表就会“断裂”或“错乱”。

<font color="#d83931">总结：</font>

|**操作 (Operation)**|**单向链表 (Singly)**|**双向链表 (Doubly)**|**备注**|
|---|---|---|---|
|**内存占用**|低 (1个指针/节点)|高 (2个指针/节点)||
|**反向遍历**|**不可能**|O(n)||
|访问第 k 个元素|O(n)|O(n)|都必须从头走。|
|在头部插入|O(1)|O(1)||
|在头部删除|O(1)|O(1)||
|在尾部插入 (有 `Tail` 指针)|O(1)|O(1)||
|**在尾部删除 (有 `Tail` 指针)**|**O(n)**|**O(1)**|**核心考点！**|
|在中间插入/删除|O(n)|O(n)|必须先O(n)“找到”那个位置。|

### 2-5 在<span style="background:#fdbfff">单链表</span>中，删除 p 所指结点的后继结点，其语句应该为 ▁▁▁▁▁ 。
A.s = p; p->next = s;

B.s = p; p->next = s->next;

C.s = p->next; p->next = s;

<span style="background:#fff88f">D.s = p->next; p->next = s->next;</span>

##### <span style="background:#d2cbff">在C语言中，我们用“指针”来操作链表：</span>
将指针理解为每节“车厢”的<span style="background:rgba(136, 49, 204, 0.2)">“乘务员”</span>，假设我们有三节车厢：
`... -> [车厢 A] -> [车厢 B] -> [车厢 C] -> ...`
乘务员p正指向`[车厢 A]`,我们的目标是删除`p`的”后继“，也就是`[车厢 B]`。
###### 怎么删除 [车厢 B]？
- 我们必须“绕过” (Bypass) 它。
- 我们必须把 **[车厢 A]** 的 `next` 挂钩，**解开**，然后**直接连到 [车厢 C]** 上。
<font color="#d83931">操作图</font><font color="#d83931">：</font>
```text
... -> [车厢 A] -> [车厢 B] -> [车厢 C] -> ...
        ^ (p 指着)->(要删除)->(B的下一个)
        |                     ^
        +------(p->next)------+
                （把 A 的挂钩改了，连到 C）
```

###### `s = p->next`; 
`s`是第二个“乘务员”，`p->next`是指`p`作为第一个乘务员指向[车厢A]的`next`的挂钩；`s = p->next`的意思是派第二个乘务员去到[车厢B]那儿
###### `p->next = s->next`;
- `s->next`的意思是第二个乘务员此时在[车厢B]的`next`挂钩上；
- `p->next`的意思是第一个乘务员所在的[车厢A]的`next`挂钩；
- `p->next = s->next`的意思就是把[车厢A]与[车厢C]连接起来

**执行后状态 (Bypass 完成！)：** `... -> [车厢 A (p 指着)] --- (新挂钩) ---> [车厢 C] -> ...`

###### <span style="background:#d2cbff">我们为什么需要两个乘务员？</span>

1. `p` (接线员)：
    - 站在被删节点(B)的前一个(A)。
        
    - 它的任务是**“绕过”** B，把 A 的 `next` 挂钩直接连到 C。
    
2. `s` (清理工)：
    - 站在被删节点(B)上。
        
    - 它的任务是<font color="#5f497a">“记住”B 的位置</font>。
        
    - **目的：** 否则 `p` 一旦“接线”完成，B 就会变成“幽灵”<span style="background:rgba(240, 167, 216, 0.55)">（内存泄漏）</span>。`s` 必须记住 B 的地址，以便在最后**销毁 (`free`)** 它。
总结代码：
```c
// 状况: p 指向 [A], 我们要删除 [B]

// 1. "s 记住 B"
s = p->next; 

// 2. "p 绕过 B, A 连接 C"
p->next = s->next; 

// 3. "s 销毁 B"
free(s);
```

### 2-7 在包含 n 个数据元素的<font color="#8064a2">单链表</font>中，▁▁▁▁▁ 的时间复杂度不为 O(n)。

A.访问第 i(1≤i≤n) 个数据元素

B.在位序 i(1≤i≤n+1) 处插入一个新结点

C.删除位序 i(1≤i≤n) 处的结点

<span style="background:#fff88f">D.将 n 个元素按升序排序</span>

##### 为什么 D 不是 O(n)，也不是 O(log n)？
<span style="background:rgba(236, 57, 57, 0.55)">排序 (Sorting) 是一个远比“访问一次”复杂得多的操作。</span>
######  为什么不是 O(log n)？
- **$O(\log n)$ 意味着什么？**
    
    - $O(\log n)$ 是一种**极其快**的““**查找**””速度。它意味着你**每一步都能排除掉一半的数据**。最典型的例子是<span style="background:rgba(136, 49, 204, 0.2)">“二分查找” (Binary Search)</span>。
    - **前提：**<span style="background:rgba(136, 49, 204, 0.2)"> 二分查找必须能“随机访问”（Random Access）</span>，也就是能**瞬间跳到**任何一个位置（比如 `arr[50]`）。
- **链表的致命伤：**
    - <span style="background:rgba(236, 57, 57, 0.55)">链表*不能*随机访问！</span>你永远无法“跳到”中间。
    - 排序时，你**至少**要把**每一节车厢都看一遍**吧？你光是“看一遍”所有车厢，这个操作**本身**就已经是 **O(n)** 了。
    - 所以，任何<font color="#d83931">链表的排序算法的**时间复杂度下限**（最快可能）都是 **O(n)**</font>。$O(\log n)$ 连看都看不完，根本不可能排好序。
###### 它的真实复杂度是什么？
这取决于你用什么排序算法。

**1. 最优的算法 (Merge Sort / 归并排序)：$O(n \log n)$**

- 这是在链表上能做到的**最快**的“比较类”排序算法。
    
- 它会把链表拆分成很多小段，分别排好序，再合并起来。
    
- **$O(n \log n)$** 是一个**非常经典**的排序复杂度。它比 $O(n)$ 慢，但比 $O(n^2)$ 快得多。

**2. 简单的算法 (Insertion Sort / 插入排序)：$O(n^2)$**

- **操作：**
    - 抓出第 1 个元素，它自己是“已排序”的。
        
    - 抓出第 2 个元素，**遍历**“已排序”的部分，把它插到正确位置。
        
    - 抓出第 3 个元素，**遍历**“已排序”的部分，把它插到正确位置。
        
    - ...
        
- **最坏情况：** 每次插入都要“遍历”一遍所有“已排序”的元素。这是一个嵌套循环。
- **时间复杂度：** $O(n \times n)$ = **$O(n^2)$**

**结论：**

- 排序的时间复杂度**至少**是 **$O(n \log n)$** 或 **$O(n^2)$**。
    
- 无论哪一个，它们都**不等于 O(n)**。
    
- 所以 D 选项的时间复杂度“**不为 O(n)**”。

这道题考察的就是你是否能一眼区分开：

- **A, B, C：** <span style="background:rgba(240, 167, 216, 0.55)">“一次遍历”操作 -> `O(n)`</span>
    
- **D：** <span style="background:rgba(240, 167, 216, 0.55)">“排序”操作 -> O(n log n) 或 O(n^2)</span>

### 2-8 单链表结点定义为（data,next）,指针H为<span style="background:rgba(236, 57, 57, 0.55)">带</span><span style="background:rgba(236, 57, 57, 0.55)">头结点</span>的<span style="background:rgba(74, 82, 199, 0.2)">单链表头指针</span>，判定带头结点单链表H为<span style="background:rgba(136, 49, 204, 0.2)">空单链表</span>的是
A.H == NULL<font color="#ccc1d9">（不带头结点）</font>

B.H->next !=NULL
   
C.H->data == 0

<span style="background:#fff88f">D.H->next == NULL</span>
##### 什么是<span style="background:rgba(236, 57, 57, 0.55)">“带头结点”</span>？(用“火车”比喻)

###### 不带头结点：
- `H` (头指针) 是一个“乘务员”。
- 如果火车**有车厢**：`H` 就指向**第一节 [数据车厢]`。
- 如果火车**是空的**：`H` 指向 `NULL` (空轨道)。
- **判空条件：`H == NULL`**
###### 带头结点：
- “头结点”是一个<span style="background:#fdbfff">特殊的、永久存在的、不装货物 (data)</span> 的 **“火车头 (Engine)”**。
- <span style="background:rgba(236, 57, 57, 0.55)">`H` (头指针) 这个“乘务员”，**永远**指向这个 `[火车头]`。</span>
- **所有真正的 [数据车厢]**，都挂在 `[火车头]` 的**后面**。

---
###### 在“带头结点”的新规矩下，“空链表”长什么样？

- “空链表”的意思是<span style="background:#b1ffff">“没有数据车厢”</span>。
    
- 但是！按照新规矩，即使没有数据车厢，那个**永久的 `[火车头]` 还是在的**！
    
- 所以，“空链表”长这样：

`H (乘务员)` -> `[火车头 (H 指向的节点)]` -> `NULL (后面没挂车厢)`

---

###### 用“新规矩”来分析所有选项
- `H`：是指向 `[火车头]` 的指针。

- `H->next`：是 `[火车头]` 的 `next` 挂钩。

- `H->data`：是 `[火车头]` 里的“货物”（我们不关心，它是垃圾数据）

**A. `H == NULL`**

- **翻译：** “`H` 乘务员指向 `NULL` (空轨道)”。
    
- **为什么错：** 在新规矩下，`H` 乘务员**永远**指向 `[火车头]`，他永远不为 `NULL`。（除非你连火车头都没造出来，那另当别论）。
    
- **(陷阱)：** 这是“不带头结点”**的判空条件。

**B. `H->next != NULL`**

- **翻译：** “`[火车头]` 的 `next` 挂钩**不**指向 `NULL`”。
    
- **为什么错：** 这说明 `[火车头]` 后面**挂着东西**（有数据车厢）！这是“链表不为空”**的判断条件。

**C. `H->data == 0`

- **翻译：** “`[火车头]` 里的货物等于0”。
    
- **为什么错：** “头结点”的 `data` 域是**无意义的、被废弃的**。我们永远不应该用它来判断任何事。

**D. `H->next == NULL`**

- **翻译：** “`[火车头]` 的 `next` <h3></h3>挂钩**正**指向 `NULL`”。
- **为什么对：** 这完美地描述了我们上面画的“空链表”状态： `H -> [火车头]` -> `NULL`
- 这说明 `[火车头]` 后面**没有**挂任何 [数据车厢]。这**就是**“带头结点的空链表”。

### 2-9 <span style="background:rgba(236, 57, 57, 0.55)">循环链表</span>的主要优点是（）。

A.不再需要头指针了

B.已知某个结点的位置后，能够很容易找到它的直接前驱
<font color="#ccc1d9">O(n)</font>

C.在进行插入、删除运算时，能更好的保证链表不断开
<font color="#ccc1d9">（无论哪种链表，你只要在操作 `p->next = s->next;` 时手滑，忘记了其中一步，链表都会“断开”（指针丢失）。这和它是不是“循环”的没关系，只和你的代码逻辑是否严谨有关。）</font>

<span style="background:#fff88f">D.从表中的任意结点出发都能扫描到整个链表</span>

##### <span style="background:rgba(236, 57, 57, 0.55)">循环链表</span>:
###### 1.什么是循环链表？
- **普通单链表：**
    
    - 像一列**有终点**的火车。
        
    - `Head -> [车厢 1] -> [车厢 2] -> [车尾 3] -> NULL (轨道终点)`
        
    - 最后一节车厢的 `next` 挂钩指向 `NULL`（空）。
        
- **循环单链表：**
    
    - 像一个<font color="#d83931">“环形过山车</font><font color="#d83931">”</font>或者<font color="#d83931">“旋转木马”</font>**。
        
    - 它的 **“车尾”** 和 **“车头”** 被连接在了一起。
        
    - `Head -> [车厢 1] -> [车厢 2] -> [车厢 3]`
        
    - `^ |`
        
    - `|________________________|` (3号车厢的 `next` 挂钩指向 1号)
        
    - <span style="background:rgba(240, 107, 5, 0.2)">它没有 `NULL` 终点！ 它是一个闭环。</span>
###### 2.循环链表的特点：**从表中的任意结点出发都能扫描到整个链表**
- **翻译：** 无论你从“环形过山车”的哪一节车厢（任意节点）坐上去，只要你**一直“坐下去”**（顺着 `next` 走），你**一定能**经过**所有的**车厢，并最终**回到**你的出发点。
    
- **这就是它的核心定义和最大优点！**


### 2-10 <span style="background:rgba(240, 167, 216, 0.55)">带头结点的循环单链表</span>（头指针为head）为空的判定条件是（）

A. head == NULL

B.head->next == NULL

<span style="background:#fff88f">C.head->next == head</span>

D.head!=NULL

##### 因为“循环”二字，彻底“杀死”了 `NULL`！

- **普通链表：** `NULL` = 结束的标志。
    
- <span style="background:#d2cbff">循环链表： `head` = 结束/开始的标志</span>。
所以：
- **普通空链表 (带头)：** `head->next == NULL` (车头指向终点)
    
- **循环空链表 (带头)：** `head->next == head` (车头指向自己)

### 2-11 在<span style="background:rgba(236, 57, 57, 0.55)">双向循环链表</span>结点`p`之后插入`s`的语句是：

A.`p->next=s; s->prior=p; p->next->prior=s ; s->next=p->next;`

B.`p->next->prior=s; p->next=s; s->prior=p; s->next=p->next;`

C.`s->prior=p; s->next=p->next; p->next=s; p->next->prior=s;`

<span style="background:#fff88f">D.s->prior=p; s->next=p->next; p->next->prior=s; p->next=s;</span>
##### <span style="background:rgba(240, 167, 216, 0.55)"><span style="background:rgba(236, 57, 57, 0.55)">双向</span>循环链表</span>：
它是一个**双向**的**环形地铁**。
- **双向：** 每一节车厢 `[S]`，既有 `s->next` 指向**下一站**，也有 `s->prev` 指向**上一站**。
- **循环：**
    
    - `[车头]` 的 `prev` 挂钩，不再指向 `NULL`，而是指向 **`[车尾]`**。
        
    - `[车尾]` 的 `next` 挂钩，不再指向 `NULL`，而是指向 **`[车头]`**。
<span style="background:#d2cbff">它是一个“无缝”的、双向的“环”，没有任何 NULL</span>。
- **双向循环链表：** 结合了“双向”和“循环”的最终形态。
- **双向插入/删除：** <span style="background:rgba(240, 107, 5, 0.2)">必须**修改 4 个指针**。</span>
###### 结点p之后插入s的操作流程：
1. 最初：
	`... <-> [车厢 p] <-> [车厢 p_next] <-> ...`

说明：
- `[车厢 p_next]` 是 `p` 的“下一个”车厢，也就是 `p->next`。
- `[车厢 p]` 和 `[车厢 p_next]` 之间，有两根挂钩连着。
- `[车厢 s]` 是一个“孤儿”，在外面飘着。

2. 目标：
	我们要把 `s` 插入到 `p` 和 `p_next` 之间，变成这样：
	`... <-> [车厢 p] <-> [车厢 s] <-> [车厢 p_next] <-> ...`

3. 接线：
	新车厢`s`为中心来思考：
- **`s` 的 `prev`：** `s` 的“前向”挂钩 (`prev`) 必须连到 `p`。
    
    - **代码：`s->prior = p;`** (`prior` 和 `prev` 都是“前一个”的意思)
        
- **`s` 的 `next`：** `s` 的“后向”挂钩 (`next`) 必须连到 `p_next`。
    
    - `p_next` 是谁？它就是 `p` 原来的 `next`。
        
    - **代码：`s->next = p->next;`**
- **`p` 的 `next`：** `p` 的“后向”挂钩 (`next`) 必须**从 `p_next` 改签**，连到 `s`
    - **代码：`p->next = s;`**
        
- **`p_next` 的 `prev`：** `p_next` 的“前向”挂钩 (`prev`) 必须**从 `p` 改签**，连到 `s`
	- **代码：`p->next->prior = s;`** (因为 `p_next` 就是 `p->next`)

###  <font color="#c00000">2-12</font> 在<font color="#d99694">单链表</font>中，将 s 所指新结点插入到 p 所指结点之后，其语句应该为 ▁▁▁▁▁ 。

A.p->next = s; s->next = p->next;

<span style="background:#fff88f">B.s->next = p->next; p->next = s;</span>

C.s->next = p->next; p->next = s->next;

D.p->next = s->next; s->next = p->next;
##### “挂钩”的本质：它是一个“槽位”
`p->next` ( `[Car P]` 的 `next` 挂钩) 就像一个**只能插一个插头**的“插座”。

- **(Before)** 它的“插座”里**正插着** `[Car P_Next]` (车厢B) 的“插头”。
##### 错误操作：`p->next = s;`
- `p->next = ...`
    
    - 这个动作是：“`[Car P]`！我要**拔掉**你屁股上**现在插着**的那个插头！”
        
- `= s;`
    
    - “然后，你把 `[Car S]` (车厢S) 的“插头”**插进去**！”
        

**发生了什么？**

1. `[Car P]` (车厢A) **拔掉了** `[Car P_Next]` (车厢B) 的插头。
    
2. `[CarP]` (车厢A) **插上了** `[Car S]` (车厢S) 的插头。

**结果 (灾难)：**

- **`[Car P_Next]` (车厢B)怎么样了？**
    
    - 它被**拔掉了**。
        
    - 没有任何车厢（`P` 或 `S`）**记得** `[Car P_Next]` (车厢B) 的位置了！
        
    - 它和它后面的**所有车厢**（C, D, E...）都**永远地丢失了**！
        
- **这就是“链表断了”的真正含义。**
    
    - 你**不是**把 S 连到 `p->next` 后面。
        
    - 你是**用 S 替换了 `p->next`**。
    
---

##### <span style="background:rgba(236, 57, 57, 0.55)">正确的操作：`s->next = p->next;`</span>
- `s->next = ...`
    - 这个动作是：“`[Car S]`！你现在是个‘孤儿’，你屁股上的‘插座’是空的。”
        
- `... = p->next;`
    
    - “你**不要拔掉** `[Car P]` 上的插头！你只是**复制**一下那个插头（`p->next`）上面的“地址”，然后也去连接 `[Car P_Next]` (车厢B)！”
**发生了什么？**
1. `[Car S]` (车厢S) 的屁股，**也连上了** `[Car P_Next]` (车厢B)。
    
2. `[Car P]` (车厢A) 的屁股，**还连着** `[Car P_Next]` (车厢B)。
    
    - (此时 A 和 S **都**指向 B)

**然后执行第二步：`p->next = s;`**

1. `[Car P]` (车厢A) **拔掉了** `[Car P_Next]` (车厢B) 的插头。
    
2. `[Car P]` (车厢A) **插上了** `[Car S]` (车厢S) 的插头。

**结果 (完美)：**

- `[Car P]` (A) 指向 `[Car S]` (S)。
    
- `[Car S]` (S) 指向 `[Car P_Next]` (B)。
    
- `A -> S -> B`。**没有任何车厢丢失！**

**总结：** 你必须先用 `s->next = p->next;` **“备份”** 那个 `p->next` 的连接，**然后**才能用 `p->next = s;` **“覆盖”** 它。

### 2-13 在<span style="background:rgba(236, 57, 57, 0.55)">双链表</span>中，删除s所指结点，其语句应该为____________。

<span style="background:#fff88f">A.s->next->prev = s->prev; s->prev->next = s->next;</span>

B.s->next = s->next->next; s->next->prev = s;

C.s->prev->next = s; s->prev = s->prev->prev;

D.s->prev = s->next->next; s->next = s->prev->prev;

<font color="#de7802">本题是2-11 题（双向循环插入）的“反向”操作</font>
##### <span style="background:rgba(240, 167, 216, 0.55)">双向循环链表是<font color="#d83931">双链表</font>首尾相连的特殊形式</span>
###### 1. 拆解题目：“双向地铁”拆车厢
- **“在双链表中”**：我们又坐上了“双向地铁”。<span style="background:rgba(136, 49, 204, 0.2)">每节车厢都有 `next` (向后) 和 `prev`/`prior` (向前) 两个挂钩。</span>
- **“删除 s 所指结点”**：`s` 乘务员正指向我们要“拆掉”的那节车厢 `[Car S]`。
---
###### 2. 透彻解析：“Bypass 绕过” S

###### 状况 (Before)：
地铁是这样连的： `... <-> [Car P] <-> [Car S] <-> [Car N] <-> ...`

- `[Car S]`：是 `s` 指向的“待删除车厢”。
    
- `[Car P]`：是 `s` 的**前一节** ( `s->prev` )。
    
- `[Car N]`：是 `s` 的**后一节** ( `s->next` )。
###### 目标 (After)：
我们要把 `[Car S]` 彻底“拆走”，让 `[P]` 和 `[N]` 直接连在一起： 
`... <-> [Car P] <-> [Car N] <-> ...`
###### 3. 思考：为了达到这个目标，我们需要修改几个挂钩？
**答案：两个！**
1. `[Car P]` 的 `next` 挂钩：
    
    - 它**原来**指向 `[Car S]`。
        
    - 它**必须**“绕过” S，**改**为指向 `[Car N]`。
        
2. `[Car N]` 的 `prev` 挂钩：
    
    - 它**原来**指向 `[Car S]`。
        
    - 它**必须**“绕过” S，**改**为指向 `[Car P]`。

只要这两根线接好了，`[Car S]` 就会被完美地“Bypass”掉。
