# 数据类型转换 (Data Type Conversion)

## 1. 自动类型转换 (Auto Casting) 
> **口诀**：小杯倒进大杯，安全！
> **规则**：数据范围小的类型，可以直接赋值给范围大的类型，系统自动完成。

###  转换链条 (必背图谱)
根据 PPT 图示，晋升路线如下：
`byte` $\rightarrow$ `short` $\rightarrow$ **`int`** $\rightarrow$ `long` $\rightarrow$ `float` $\rightarrow$ **`double（最高级）`**
	           $\nearrow$
         `char`

* **特例**：`char` 可以自动变成 `int`（查 ASCII 码表），但<span style="background:rgba(240, 107, 5, 0.2)">不能</span>自动变成 `short` 或 `byte`（<span style="background:rgba(240, 107, 5, 0.2)">因为 char 没有负数，而 short/byte 有负数</span>，范围不完全重合）。
### 考试防坑实例
```java
int a = 100;
double b = a;  // ✅ 正确：int 自动变 double
System.out.println(b); // 输出 100.0 (注意带小数位)

char c = 'A';
int d = c;     // ✅ 正确：char 自动变 int (65)
```

## 2. 强制类型转换 (Explicit Casting) 

> 口诀：<span style="background:rgba(236, 57, 57, 0.55)">大杯倒进小杯，必须手动“削减”！</span>
> 
> 风险：可能会 丢失精度 或 发生溢出（数据变乱）。

### ⚔️ 语法格式

```JAVA
目标类型 变量名 = (目标类型) 原值;
```

### 两个经典后果 (考试必考)

1. **丢失精度 (截断小数)**
    
    - 把浮点数强转为整数，**直接丢弃小数部分**（<font color="#ff0000">不是四舍五入！</font>）。
    
```Java
  double d = 3.99;
  int i = (int) d; // 结果是 3，不是 4！
```
    
2. **数据溢出 (二进制截断)**
    
    - **现象**：大整数强转进 byte，可能变成负数！
        
    - **原理**：砍掉了前面的二进制位，导致符号位变化。
    
```Java
    int n = 129;
    byte b = (byte) n; 
    // 😱 结果是 -127 (课件原例！)
    // 解释：129 二进制是 00...10000001
    // 强转 byte 只留后8位 -> 10000001 (首位是1，变成负数补码)
    ```

## 3. 表达式中的“潜规则” (Promotions) 

> **定义**：当不同类型的数据在一起做 `+ - * /` 运算时，结果是什么类型？

### 规则 A：全员升职记

只要表达式中有一个是“老大”，全员都得陪着变成“老大”。

- 有 `double` $\rightarrow$ 结果是 `double`
    
- 没 double 有 `float` $\rightarrow$ 结果是 `float`
    
- 没 float 有 `long` $\rightarrow$ 结果是 `long`
### 规则 B：底层保底 (The Int Rule)

这是**最容易扣分**的陷阱！

- **`byte`、`short`、`char` 只要参与运算，不管有没有跟别人混，自己先自动提升为 `int` 再算！**

### 考试防坑实例

```Java
byte b1 = 10;
byte b2 = 20;
// byte b3 = b1 + b2; // ❌ 报错！
// 原因：b1 和 b2 运算时都变成了 int，结果也是 int。
// 正确写法：
byte b3 = (byte)(b1 + b2); // 强转回来
int b4 = b1 + b2;          // 或者直接用 int 接

char c = 'A'; // 65
System.out.println(c + 1); // 输出 66 (int类型)
```

## 4. 字符串的“黑洞效应” (String Concatenation) 

> 规则：String + 任何东西 = String。
> 字符串具有极强的“传染性”，会把<span style="background:rgba(240, 107, 5, 0.2)">后面连接的东西都变成字符串</span>。

### 考试防坑实例

```Java

System.out.println("Result: " + 5 + 5); 
// 输出 "Result: 55" (先变成 "Result: 5"，再接个 "5")

System.out.println(5 + 5 + "Result");
// 输出 "10Result" (先算 5+5=10，再接字符串)
```


# 课后习题详解 (数据类型与转换)

## 习题 3：判断表达式能否正确执行

> **题目背景**：已知 `int a = 10; long b = 1;`
> **考察点**：类型转换规则、赋值语句的左值要求。

| 序号 | 题目表达式 | 是否正确 | 💡 教授解析 (核心考点) |
| :--- | :--- | :--- | :--- |
| 1) | `a = b;` | ❌ **错** | **大转小**：`b`是long，`a`是int。大杯倒小杯必溢出，必须强转 `(int)b`。 |
| 2) | `b = a;` | ✅ **对** | **小转大**：`a`是int，`b`是long。安全，自动类型提升。 |
| 3) | `b = a + b;` | ✅ **对** | `a+b` 的结果自动升级为 `long`，赋给 `long` 类型的 `b`，类型匹配。 |
| 4) | `a = (int b);` | ❌ **错** | **语法错误**：强转括号的位置不对！应该是 `(int)b`，Java不认识 `(int b)`。 |
| 5) | `a = (int)b;` | ✅ **对** | **强制类型转换**：标准写法。虽然可能丢失精度，但语法正确。 |
| 6) | `(int)a = b;` | ❌ **错** | **左值错误**：等号左边必须是**变量名**！`(int)a` 是一个算出来的**数值**，不能给数值赋值。 |
| 7) | `b = (long)a;` | ✅ **对** | 虽然 `int` 转 `long` 是自动的，但你手动加强转也是合法的（多此一举但没错）。 |
| 8) | `(int)b = a;` | ❌ **错** | **左值错误**：同第6题，不能给强转后的结果赋值。 |

## 习题 4：判断表达式能否正确编译

> **考察点**：浮点数的后缀 (`f`, `d`)、默认类型 (`double`)、整型赋值规则。

| 序号  | 题目表达式                    | 是否正确    | 💡 教授解析 (核心考点)                                                                                             |
| :-- | :----------------------- | :------ | :--------------------------------------------------------------------------------------------------------- |
| 1)  | `float a = 10.5;`        | ❌ **错** | **默认Double陷阱**：`10.5` 默认是 `double`，直接给 `float` 会报错，<span style="background:rgba(236, 57, 57, 0.55)">必须加 f。 |
| 2)  | `double a = 10.5;`       | ✅ **对** | 标准写法。                                                                                                      |
| 3)  | `double a = 10.5d;`      | ✅ **对** | 显式加 `d` 表示 double，没问题。                                                                                     |
| 4)  | `double a = 10.5D;`      | ✅ **对** | 同上，大写 `D` 也可以。                                                                                             |
| 5)  | `long a = 10;`           | ✅ **对** | `10` 是 `int`，赋给 `long` 会自动提升。                                                                              |
| 6)  | `long a = 10L;`          | ✅ **对** | 标准写法。加 `L` 显式声明是 long。                                                                                     |
| 7)  | `float a = (float)10.5;` | ✅ **对** | **暴力强转**：虽然 `10.5` 是 double，但前面加了 `(float)` 强制转换，编译器放行。                                                    |
| 8)  | `float a = 10.5F;`       | ✅ **对** | 标准写法。加 `F` 告诉编译器这是 float。                                                                                  |

---

### 考前复习口诀 (必背)
1.  **赋值看左边**：等号左边只能是变量名 `a`，不能是运算结果 `a+b` 或强转 `(int)a`。
2.  **小数看后缀**：没后缀默认 `double`，想变 `float` 必须加 `f`。
3.  **大转小必报错**：除非加 `(类型)` 强转。
