# 三目与扩展运算符

## 1. 三目运算符 (Ternary Operator) 
> **别名**：三元运算符。
> **本质**：就是 **偷懒版的 `if-else`**。
> **口诀**：**"真前假后"** ——<span style="background:rgba(236, 57, 57, 0.55)"> 问号前面是条件，真的走前面，假的走后面。</span>

###  语法结构
```java
// 格式
数据类型 变量 = (条件表达式) ? 结果1 : 结果2;

// 实例
int a = (4 < 5) ? 3 : 8; 
// 1. 判断 (4 < 5) -> true
// 2. 选冒号前面的 3
// 3. 结果 a = 3
```
### 考试防坑细节
- **类型要兼容**：`结果1` 和 `结果2` 最好是同一种数据类型，或者能自动转换
- **嵌套地狱**：虽然允许 `a?b:(c?d:e)`，但考试如果出这种题，请从 **最里面** 的问号开始拆解。

## 2. 扩展赋值运算符 (Extended Assignment) 

> **定义**：将算术运算和赋值运算合二为一。 **符号**：`+=`, `-=`, `*=`, `/=`, `%=`

### 运行机制

- `a += b` 等价于 `a = a + b`
    
- `a *= b` 等价于 `a = a * b`

###  顶级隐藏考点：自带强转 (Implicit Casting)

这是 **这一页 PPT 最容易被忽略** 但 **最爱考** 的点！

```Java
short s = 1;

// ❌ 写法 A (报错)
// s = s + 1; 
// 原因：1是int，s+1 变成 int，赋给 short 类型的 s 会报错。

// ✅ 写法 B (正确)
s += 1; 
// 原因：==扩展运算符内部自动包含了强制类型转换==！
// 等价于：s = (short)(s + 1);
```

## 3. 运算符优先级 (Precedence) 

**从高到低 (High -> Low)：**

1. 👑 **第一梯队 (括号)**：`()` —— 有括号先算括号，无敌。
    
2. ⚔️ **第二梯队 (单目)**：`++`, `--`, `!` (非) —— 贴着变量的先算。
    
3. 🧮 **第三梯队 (算术)**：`* / %` (乘除模) **>** `+ -` (加减)。
    
4. ⚖️ **第四梯队 (关系/逻辑)**：`> < ==` (比大小) **>** `&& ||` (逻辑判断)。
    
5. 🐢 **第五梯队 (赋值)**：`=` , `+=` —— 最后才给变量赋值。


###  记忆口诀

**“括号最先，单目紧随，先乘除后加减，比完大小再逻辑，最后才赋值。”**

### 📝 习题复盘 (解析你的满分思路)

#### **第 1 题：选 D (5) ✅**
> `int a=17, b=6; result=(a%b>4) ? a%b : a/b;`

1.  **算优先级**：`()` 里的 `a%b` 先算。
2.  **算取模**：`17 % 6`。$17 \div 6 = 2 \dots 5$。余数是 **5**。
3.  **算比较**：`5 > 4` 为 **true**。
4.  **三目选择**：因为是 true，选冒号前面的 `a%b`。
5.  **结果**：**5**。

#### **第 2 题：结果是 2 ✅**
> `int x=4; (x>4) ? 1 : 2;`

1.  **算比较**：`4 > 4` 为 **false** (4不大于4，是等于)。
2.  **三目选择**：因为是 false，选冒号后面的 **2**。

#### **第 3 题：选 A (排序) ✅**
> `()`, `!`, 算术, 关系, 逻辑, 赋值

这完全符合我们的 **“5 层金字塔”** 理论。
* `()` 第一。
* `!` (非运算) 是单目运算符，排第二。
* `+ - * /` 算术排第三。
* `> <` 关系排第四。
* `&&` 逻辑排第五。
* `=` 赋值永远垫底。
