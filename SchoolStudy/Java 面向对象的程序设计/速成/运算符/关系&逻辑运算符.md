# 关系与逻辑运算符 (Logic & Relational)

## 1. 关系运算符 (Comparison) 
> **作用**：比大小。结果永远只有 **`true`** 或 **`false`**。

| 符号 | 含义 | ⚠️ **考试防坑点** |
| :--- | :--- | :--- |
| `==` | 等于 | **千万别写成 `=` (赋值)！** `if(a=b)` 是死罪。 |
| `!=` | 不等于 | |
| `>` / `<` | 大于/小于 | |
| `>=` / `<=` | 大于等于... | 符号顺序不能反，`=>` 是错的。 |

### 字符串比较特例 (必考)
根据 PPT，比较两个字符串内容是否相同：
* ❌ **错误**：`s1 == s2` (比的是内存地址，不是内容)
* ✅ **正确**：`s1.equals(s2)` (比的是字面上的字)

## 2. 逻辑运算符 (Logical) 
> **作用**：连接多个条件。比如“既要...又要...”。

| 符号       | 名称      | 逻辑口诀          | 特性                |
| :------- | :------ | :------------ | :---------------- |
| **`&&`** | **短路与** | **全真才真，一假即假** | ⚡️ **带短路功能** (常用) |
|  \|\|    | 短路或     | **一真即真，全假才假** | ⚡️ **带短路功能** (常用) |
| `!`      | 非       | 真变假，假变真       | 取反                |
| `&`      | 逻辑与     | 全真才真          | ❌ 不短路 (笨，两边都算)    |
| \|       | 逻辑或     | 一真即真          | ❌ 不短路 (笨，两边都算)    |
| `^`      | 异或      | **不同为真，相同为假** | (男男为假，男女为真)       |

## 3. 核心考点：什么是“短路” (Short Circuit)？ ⚡️
这是本节最难理解、但考试出题率 100% 的点！

### A. 短路与 (`&&`) 的“偷懒”规则
> **规则**：如果**左边**已经是 `false` 了，整个式子铁定是 `false`。
> **后果**：计算机**直接跳过右边**，右边的代码（哪怕是 `++` 自增）**根本不会执行**！

```java
int a = 1;
// 左边 (6 < 5) 是 false。
// 计算机心想：反正结果也是假，后面不用看了。
// 结果：++a 被跳过，a 还是 1。
boolean b = (6 < 5) && (++a > 1); 

System.out.println(a); // 输出 1 (没变！)
```

### B. 短路或 (`||`) 的“偷懒”规则

> **规则**：如果**左边**已经是 `true` 了，整个式子铁定是 `true`。 **后果**：计算机**直接跳过右边**。

```Java
int a = 1;
// 左边 (6 > 5) 是 true。
// 计算机心想：结果肯定是真，收工！
boolean b = (6 > 5) || (++a > 1);

System.out.println(a); // 输出 1 (没变！)
```

## 4. `&&` vs `&` 的区别 (面试/考试必问)

- **`&&` (短路)**：聪明。左边定生死，右边可能不执行。
    
- **`&` (非短路)**：死板。不管左边是什么，右边**一定**会执行。
    
- **结论**：考试代码题里，看到 `&&` 就要警惕右边的 `++` 有没有被跳过！

### 习题超详细解析 (解救你的困惑)

现在回到让你头大的习题 1。
xs
**题目**：
```java
int x = 6, y = 8;
boolean b;
// 核心代码：
b = x > y && ++x == --y;
````

求 `b`, `x`, `y` 的值。

**教授的拆解步骤：**

1. **看左边**：`x > y` 即 `6 > 8`。
    
    - 结果是 **`false`**。
        
2. **看符号**：中间是 **`&&` (短路与)**。
    
    - 根据口诀“一假即假”，<span style="background:#fff88f">既然左边已经是 `false`，那么整个表达式的结果已经注定是 `false`。</span>
        
    - **⚡️ 触发短路机制！**
        
3. **看右边**：`++x == --y`。
    
    - 由于<span style="background:rgba(236, 57, 57, 0.55)">发生了短路</span>，**这一坨代码直接被无视！** 计算机根本没有运行它。
        
    - 所以，`x` 没有加 1，`y` 也没有减 1。
        
4. **最终结果**：
    
    - **`b`** = **`false`** (你可能以为 b 是数字，但 b 是 boolean 类型)。
        
    - **`x`** = **`6`** (保持原样)。
        
    - **`y`** = **`8`** (保持原样)。
        

**❌ 你的错误原因**：你算出了 `x=7`，说明你虽然判断对了逻辑，但你没刹住车，把右边的 `++x` 给执行了。记住：**看到 `&&` 左边为假，右边立刻“死”掉！**

**题目 2**：

```Java
x = 5, y = 10;
// 表达式：
x > y && x++ == y--
```

**解析**：

1. **左边**：`5 > 10` 为 **`false`**。
    
2. **中间**：**`&&`** 短路。
    
3. **右边**：不执行。
    
4. **结果**：整个表达式的值为 **`false`**。
