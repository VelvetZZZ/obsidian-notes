# 位运算符 (底层魔法)

## 1. 计算机里的数字怎么存？ (必考基础) 
> **核心铁律**：计算机内存里，所有的数字都是以 **二进制补码 (Two's Complement)** 的形式存储的！

### A. 三码合一 (正数)
* <span style="background:#ff4d4f">正数</span>的 原码 = 反码 = 补码。
* **符号位** <span style="background:rgba(240, 107, 5, 0.2)">(最高位) 是 0</span>。
* *例*：`int 5` (假设8位简化版) $\rightarrow$ `0000 0101`
### B. 麻烦的负数 (负数) 
* **符号位** (最高位) 是 **1**。
* **原码**：人类能看懂的二进制 (`-5` $\rightarrow$ `1000 0101`)
* **反码**：符号位不变，其他位取反 (`1` 变 `0`, `0` 变 `1`)。
* **补码**：反码 + 1 (这才是计算机真正存的东西！)。

### 考试计算流程 (负数运算)
如果要算 `-5` 的二进制：
1.  写出 `5`：`0000 0101`
2.  取反：`1111 1010`
3.  加 1：`1111 1011` (这就是内存里的 -5)

## 2. 运算符大乱斗 (Operators) 
>[!WARNING] `0` 是假，`1` 是真。

| 符号      | 名称       | 规则口诀          | 例子 (4 vs 3)                             |
| :------ | :------- | :------------ | :-------------------------------------- |
| **`&`** | **按位与**  | **全1则1，有0则0** | `100 & 011` $\rightarrow$ `000` (0)     |
| \|      | **按位或**  | **有1则1，全0则0** | `100` \| `011` $\rightarrow$ `111` (7)  |
| `~`     | **按位非**  | 0变1，1变0       | `~5` $\rightarrow$ `-6` (记住公式 ==-n-1==) |
| **`^`** | **按位异或** | **相同为0，不同为1** | `100 ^ 011` $\rightarrow$ `111` (7)     |

## 3. 移位运算符 (Shift) 
> 这里的重点是：**符号位怎么办？**

| 符号        | 名称        | 动作           | 符号位处理              | 效果 (数学)      |
| :-------- | :-------- | :----------- | :----------------- | :----------- |
| **`<<`**  | 左移        | 往左推，==右边补0== | 不关心                | $\times 2^n$ |
| **`>>`**  | **带符号右移** | 往右推          | **正补0，负补1** (保持正负) | $\div 2^n$   |
| **`>>>`** | **无符号右移** | 往右推          | **统统补0** (变大正数)    | 变成巨大的正数      |

### 考试代码详解 (PPT 第 9 页)
```java
// 1. 左移 (乘法)
System.out.println(5 << 1); 
// 5 (0101) 左移1位 -> 1010 (10)

// 2. 带符号右移 (除法)
System.out.println(5 >> 1);
// 5 (0101) 右移1位 -> 0010 (2)

// 3. 无符号右移负数 (高能预警!) 🔥
System.out.println(-5 >>> 1);
// 内存里的 -5 (补码): 11111111...11111011 (32个位)
// 向右移 1 位，左边强行补 0:
// 结果: 01111111...11111101 
// (符号位变成了0，瞬间变成一个巨大的正数: 2147483645)
```
## 4. 异或的魔法 (XOR Magic) 🎩

> **必考公式**：`a ^ a = 0` (自己异或自己等于0) **推导**：`a ^ 0 = a`

**应用场景**：不使用临时变量交换两个数，或者消除重复数。

### 习题超详细解析 (解救你的进制转换)

#### **第 1 题：`4 | 3` (你答 1 ❌)**
* **分析**：这是 **`|` (或运算)**，不是 `&` (与)。
* **转换**：
    * 4 的二进制 $\rightarrow$ `100`
    * 3 的二进制 $\rightarrow$ `011`
* **计算** (`|` 有1就变1)：
    * `1` `0` `0`
    * `0` `1` `1`
    * -----------
    * `1` `1` `1`
* **结果**：二进制 `111` 转成十进制 = $4+2+1$ = **7**。

#### **第 2 题：`4 & 2` (你答 0 ✅)**
* **分析**：这是 **`&` (与运算)**。
* **转换**：
    * 4 $\rightarrow$ `100`
    * 2 $\rightarrow$ `010`
* **计算** (`&` 全1才变1)：
    * `1` `0` `0`
    * `0` `1` `1` (这里2应该是010，PPT可能有误或者我写错，按2算) -> `010`
    * -----------
    * `0` `0` `0` (没有一列是两个1)
* **结果**：**0**。

#### **第 3 题：`a ^ a ^ b` (你不会 🤯)**
> **题目**：`int a=31; int b=a>>>2; System.out.println(a^a^b);`

这题有两个考点：
1.  **异或消消乐**：
    * 公式：`a ^ a = 0`。
    * 所以表达式变成了：`0 ^ b`。
    * 公式：`0 ^ b = b`。
    * **结论：这题其实就是让你求 `b` 是多少！** `a` 是多少根本没关系！

2.  **移位计算**：
    * `b = a >>> 2` (31 右移 2 位)。
    * 31 的二进制是多少？
        * $16+8+4+2+1 = 31$，所以是 `11111`。
    * 右移 2 位 (`>>> 2`)：
        * 砍掉最右边两位 `11`。
        * 剩下 `00111`。
    * `111` 是多少？
        * $4+2+1$ = **7**。

**最终答案：7**
