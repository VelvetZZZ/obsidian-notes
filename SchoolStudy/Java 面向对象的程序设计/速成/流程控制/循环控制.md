# 循环控制三剑客 (Control Flow)

## 1. 核心区别 (The Big Three) ⚔️
> 想象你在操场**跑步（循环）**，教练对你下达了不同指令：

| 关键字            | 含义       | 跑步比喻                                           | 作用范围          |
| :------------- | :------- | :--------------------------------------------- | :------------ |
| **`continue`** | **跳过本次** | "这一圈不算，**立刻回到起跑线**，开始跑**下一圈**！"                | 仅当前循环的一轮      |
| **`break`**    | **终止循环** | "别跑了！**彻底结束**训练，去休息区喝水。"                       | 整个循环结构        |
| **`return`**   | **结束方法** | "放学了！**直接回家**。" (==不管有没有循环，整个 `main` 方法都结束了==) | 整个方法 (Method) |

---

## 2. 图解执行流 (Flowchart Logic) 

### A. Break
* 遇到 `break` $\rightarrow$ **砸烂循环** $\rightarrow$ 执行循环大括号 `}` **后面** 的代码。
* **常用于**：`switch` 语句，或者死循环中找到答案想退出时。

### B. Continue
* 遇到 `continue` $\rightarrow$ **无视** 后面剩下的代码 $\rightarrow$ **直接跳** 到：
    * `while`/`do-while`：跳去 **判断条件**。
    * `for`：跳去 **步进表达式 (`i++`)**。

### C. Return
* 遇到 `return` $\rightarrow$ **杀** 死整个函数。
* 如果在 `main` 方法里写 `return`，程序直接结束运行，`return` 后面的所有代码（哪怕在循环外面）都不会执行。

---

## 3. 考试防坑指南 (Exam Traps) 
1.  **Continue + i++**：在 `while` 中，如果 `continue` 在 `i++` 之前，会导致 `i` 永远不增加 $\rightarrow$ **死循环**！(For 循环没这问题，因为它跳去 `i++` 那一行)。
2.  **Break + 嵌套**：==`break` 只能跳出 **当前这一层** 循环==。如果想跳出最外层，需要配合 **标签 (Label)** 使用（期末考得少，知道即可）。

# 控制循环习题详解 (Break/Continue)

## ✅ 第 1 题：Do-While 与 Continue 的配合

**题目来源**：

**代码片段**：

```Java
int i = 0, s = 0;
do {
    // ⚠️ 关键点：如果是偶数，i+1 然后跳过后面代码
    if (i % 2 == 0) { 
        i++; 
        continue; 
    }
    i++;        // 奇数时，i 再加 1
    s = s + i;  // 奇数时，累加 i
} while (i < 7);
System.out.println(s);
```

**选项**：(A)16 (B)12 (C)28 (D)21

**💡 教授“慢动作”解析**：

1. **第一圈 (`i=0`)**：`0%2==0` (真)。`i` 变 **1**。执行 `continue` (跳过累加，直接去检查条件)。
    
    - 检查：`1 < 7` (真) $\rightarrow$ 继续。
        
2. **第二圈 (`i=1`)**：`1%2!=0` (假)。向下走。`i` 变 **2**。`s = 0 + 2 = 2`。
    
    - 检查：`2 < 7` (真) $\rightarrow$ 继续。
        
3. **第三圈 (`i=2`)**：`2%2==0` (真)。`i` 变 **3**。`continue`。
    
    - 检查：`3 < 7` (真) $\rightarrow$ 继续。
        
4. **第四圈 (`i=3`)**：`3%2!=0` (假)。`i` 变 **4**。`s = 2 + 4 = 6`。
    
    - 检查：`4 < 7` (真) $\rightarrow$ 继续。
        
5. **第五圈 (`i=4`)**：`4%2==0` (真)。`i` 变 **5**。`continue`。
    
    - 检查：`5 < 7` (真) $\rightarrow$ 继续。
        
6. **第六圈 (`i=5`)**：`5%2!=0` (假)。`i` 变 **6**。`s = 6 + 6 = 12`。
    
    - 检查：`6 < 7` (真) $\rightarrow$ 继续。
        
7. **第七圈 (`i=6`)**：`6%2==0` (真)。`i` 变 **7**。`continue`。
    
    - **🏁 终极检查**：`7 < 7` (假！)。**循环结束**。
        
    - _(注意：<span style="background:rgba(236, 57, 57, 0.55)">虽然 i 变成了 7，但没机会执行下面的 s+=i 了，s 保持 12</span>)_。
        

**👉 正确答案：(B) 12**

## ✅ 第 2 题：For 循环步进与 Break

**题目来源**：

**代码片段**：

```Java
int sum = 0;
// ⚠️ 关键点：i 每次 +2 (1, 3, 5...)
for (int i = 1; i < 10; i = i + 2) { 
    if (i % 2 == 0) { // 只有偶数才 break
        break;
    }
    sum += i;
}
System.out.println(sum);
```

**选项**：(A)25 (B)45 (C)1 (D)0

**💡 教授“一眼”解析**：

- 循环变量 `i` 的变化序列是：1, 3, 5, 7, 9...
    
- **全是奇数！**
    
- 所以 `if (i % 2 == 0)` 这个条件 **永远不可能成立**。
    
- `break` 永远不会执行，这只是个普通的累加。
    
- **计算**：$1 + 3 + 5 + 7 + 9 = 25$。
    

**👉 正确答案：(A) 25**

## ✅ 第 3 题：Total 累加的位置陷阱

**题目来源**：

**代码片段**：

```Java
int total = 0;
for (int i = 0; i < 4; i++) {
    if (i == 1) continue; // i=1 时跳过
    if (i == 2) break;    // i=2 时自杀
    total += i;           // ⚠️ 只有没跳过、没自杀的才能加
}
```

**选项**：(A)0 (B)1 (C)3 (D)6

**💡 教授“慢动作”解析**：

1. **i=0**：没中 continue，没中 break。执行 `total += 0`。Total = 0。
    
2. **i=1**：命中 `continue`！**直接跳过** 这一轮剩下的代码（也就是跳过了 `total += i`）。Total 还是 0。
    
3. **i=2**：命中 `break`！**整个循环直接结束**。后续代码不再执行。
    
4. **结果**：Total 从头到尾只加了个 0。
    

**👉 正确答案：(A) 0**

## ！！！第 4 题：伪装成死循环的 Break

**题目来源**：

**代码片段**：

```Java
for (int i = 0; ; ) { // 中间缺省，看似死循环
    System.out.println("这是 " + i);
    break; // 🛑 只要进来了，第一件事就是自杀
}
```

**选项**：(A)语法错误 (B)死循环 (C)不输出 (D)输出:这是0

**💡 教授解析**：

- `for(;;)` 确实是死循环的写法。
    
- 但是！==第一轮循环刚打印完 "这是 0"，马上就碰到了 `break`==。
    
- `break`<span style="background:rgba(236, 57, 57, 0.55)"> 无条件终止循环</span>。
    
- 所以只活了一轮就挂了。
    

**👉 正确答案：(D) 输出:这是0**

## ！！！第 5 题：Do-While 复杂逻辑 (BOSS 题)

**题目来源**：

**代码片段**：

```Java
int i = 1, j = 10;
do {
    // ⚠️ --j 是先减再用
    if (i > --j) { 
        continue; 
    }
} while (i++ < 5); // ⚠️ i++ 是先比再加
System.out.println("i=" + i + " j=" + j); // 假设是打印这个
```

求输出结果：_________________
**💡 教授“表格法”追踪**：

|**轮次**|**初始 i, j**|**条件 i > --j**|**Continue?**|**检查 i++ < 5**|**i 变化**|**j 变化**|
|---|---|---|---|---|---|---|
|1|1, 10|1 > 9 (假)|No|1 < 5 (真)|1 -> 2|10 -> 9|
|2|2, 9|2 > 8 (假)|No|2 < 5 (真)|2 -> 3|9 -> 8|
|3|3, 8|3 > 7 (假)|No|3 < 5 (真)|3 -> 4|8 -> 7|
|4|4, 7|4 > 6 (假)|No|4 < 5 (真)|4 -> 5|7 -> 6|
|5|5, 6|5 > 5 (假)|No|5 < 5 (**假**)|5 -> 6|6 -> 5|

- **解析**：
    
    - `if` 条件里 `i` 始终小于或等于 `j`，所以 `continue` **一次都没执行过**。
        
    - `while` 条件里，虽然第五轮 `5 < 5` 是 False，但 `i++` 是**后置自增**，比较完之后 `i` 还是要加 1，所以 `i` 最后变成了 **6**。
        
- **最终结果**：
    
    - **i = 6**
        
    - **j = 5**
