# Do-While 循环语句

## 1. 核心语法 (Syntax) 
> **口诀**：**先干活，再检查**。即使条件是假的，也至少能活过第一轮。

```JAVA
do {
    // 循环体 (至少执行 1 次)
    // 比如：输入密码、交作业
} while (条件表达式); // 🛑 这是一个“有分号”的循环！

```
### 顶级考点：那个不起眼的分号

- `while(条件) { }` 后面 **没有** 分号。
    
- `do { } while(条件);` 后面 **必须有分号**！
    
    - **考试填空题**：如果看到 `while(...)` 后面直接跟了 `;`，前面肯定有个 `do`！
---

## 2. While vs Do-While (必考区别) 

这是期末考试简答题或选择题最喜欢问的区别。

| **特性**     | **While 循环**          | **Do-While 循环**                  |
| ---------- | --------------------- | -------------------------------- |
| **执行顺序**   | 先判断，再执行               | **先执行，再判断**                      |
| **最小执行次数** | **0 次** (门槛高，条件不符不让进) | ==1 次== (保底，来者是客先招待)             |
| **生活案例**   | "先看账户余额，够钱再买东西"       | "先买彩票刮一次，没中奖再继续买"                |
| **PPT案例**  | -                     | ==学生交论文，老师审核，**不合格重写** (至少交一次)== |

---

## 3. 适用场景 (Best Practice) 

- **输入验证**：要求用户输入一个正数，如果输入错了就让他一直重输。

    - 因为你 **至少** 得让用户输入一次，所以 `do-while` 是最完美的逻辑。


### 编程题速解：三位数校验 (Input Validation)

**题目**：
> 从控制台读取一个三位数。如果输入的不是三位数，就继续提示用户输入，直到输入正确为止。

**教授的逻辑导航**：
1.  **动作**：输入数字 (至少得输一次吧？用 `do`)。
2.  **条件**：什么时候需要**继续循环**（重输）？
    * 答：当输入的数字 **错误** 的时候。
    * 什么是错误的？**小于 100** 或者 **大于 999**。
3.  **代码**：把上面的逻辑翻译成 Java。

**✨ 满分标准代码 (最简便版)**：

```java
import java.util.Scanner;

public class DoWhileDemo {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n; // 变量要在循环外面声明，否则while里看不见！
        
        do {
            System.out.println("请输入一个三位数：");
            n = sc.nextInt();
            
        // 核心条件：如果是错的(小于100 或 大于999)，就回过头再来一次
        } while (n < 100 || n > 999); 
        
        System.out.println("输入正确！你输入的数字是：" + n);
    }
}
````
