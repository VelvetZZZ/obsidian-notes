# 整型 (Integer)

## 1. 四大金刚：整型分类表 
> [!] **考试口诀**：byte 是 8 位，short 是 16，int 是 32，long 是 64。
> **默认规则**：代码里随便写一个整数（如 `100`），Java 默认它是 **`int`** 类型。

| 类型        | 占用空间 | 位数 (Bit) | 表数范围 (Range)                          | 记忆关键点                                                                            |
| :-------- | :--- | :------- | :------------------------------------ | :------------------------------------------------------------------------------- |
| **byte**  | 1 字节 | 8位       | $-128 \sim 127$ ($-2^7 \sim 2^7-1$)   | **考点**：极容易溢出！`byte b = 128` 会报错。                                                 |
| **short** | 2 字节 | 16位      | $-32768 \sim 32767$                   | 平时很少用，考试偶尔考范围。                                                                   |
| **int**   | 4 字节 | 32位      | $\approx \pm 21$ 亿                    | **最常用**。整数运算的默认类型。                                                               |
| **long**  | 8 字节 | 64位      | 很大很大 ($\approx \pm 9 \times 10^{18}$) | **考点**：加<font color="#ff0000">后缀</font> `L` 或 `l`。只有超过 int 范围（约21亿）才**必须**加 `L`。 |

### 考试防坑实例
```java
byte b1 = 127;      // ✅ 正确：最大值
byte b2 = 128;      // ❌ 编译错误：超出 byte 范围 (-128 ~ 127)

long num1 = 9999999999; // ❌ 编译错误：数字太大，且没加 L，被视为 int 处理导致溢出
long num2 = 9999999999L; // ✅ 正确：加了 'L' 表示这是 long 类型
```

## 2. 进制表示法 (Literals) 

> **定义**：如何在代码中告诉计算机“我写的是几进制”。这是**单选题**重灾区。

|**进制**|**前缀 (Prefix)**|**允许的数字**|**实例 (Example)**|**十进制值**|
|---|---|---|---|---|
|**二进制**|`0b` 或 `0B`|0, 1|`int a = 0b11;`|3|
|**八进制**|`0` (零)|0 ~ 7|`int b = 010;`|8|
|**十进制**|无|0 ~ 9|`int c = 10;`|10|
|**十六进制**|`0x` 或 `0X`|0~9, A~F|`int d = 0x10;`|16|

### 考试防坑实例:

```Java
int k = 08;   // ❌ 错误！0开头是八进制，但八进制里不能有数字 '8'
int m = 0xG;  // ❌ 错误！十六进制最大是 F，没有 G

// 经典考题：下列哪行代码正确？
int x = 012;  // ✅ 八进制，等于十进制 10 (1*8 + 2)
System.out.println(x); // 输出结果是 10，不是 12！
```

## 3. 下划线分隔符 (Java 7+ 新特性) 

> 为了让长数字更容易读，可以在数字之间<u>加下划线 </u>_。
> 
> 规则：只能加在数字中间，不能加在开头、结尾或小数点旁。

```Java
int million = 1_000_000; // ✅ 合法，非常清晰
long creditCard = 1234_5678_9012_3456L; // ✅ 合法

// ❌ 错误示范
int x = _100;  // 不能开头
int y = 100_;  // 不能结尾
long z = 0x_10; // 不能紧跟前缀
```

>[!WARNING] 教授的引导: 整型这一节如果你弄懂了“溢出”和“进制前缀”，基本上就拿稳了。 接下来我们要进入一个稍微麻烦一点的领域，也是手写代码题里容易扣分的地方：**浮点型（小数）**。这里有一个经典的坑：`float f = 3.14;` 这行代码是错的，你知道为什么吗？我们要不要接着把“浮点型”也一并速通了？

# 错题：

>[!EXAMPLE] **第 1 题：下列哪个语句是不正确的？**
 **A) `float f=11.1`** ❌ **(选它！这是错的)**
 **解析**：`11.1` 默认是 `double`。你试图把它塞给 `float f`，由于没有加后缀 `f`，编译器报错。
 **D) `double f=11.1E10f`** ✅ **(这是对的)**
**解析**：虽然写法很怪，但 `11.1E10f` 表示这是一个 float 类型的科学计数法数值。把 float 赋值给 double 变量是可以的（小杯倒进大杯，自动转换）。

>[!EXAMPLE] **第 2 题：以下哪个声明是错的？**
**B) `float f=1.1`** ❌ **(选它！这是错的)**
**解析**：同上。`1.1` 是 double，不能直接给 float，必须写成 `1.1f`。
>**D) `long m=4990`** ✅ **(这是对的)**
-**解析**：你可能以为 long 后面必须加 `L`？其实不用。`4990` 是一个 `int`，它完全放得下 `long` 的空间，所以会自动提升类型。只有超过 int 范围（约21亿）才**必须**加 `L`。


# 浮点型 (Float/Double)

## 1. 浮点型分类 
> **定义**：用于表示带小数的数字。Java 遵循 IEEE 754 标准。
> **口诀**：小数默认是 double，想用 float 必须加 f。

| 类型 | 关键字 | 占用空间 | 精度 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **单精度** | `float` | 4 字节 | 约 7 位有效数字 | 只有在极度节省内存时才用 (很少用) |
| **双精度** | `double`| 8 字节 | 约 15 位有效数字 | **日常开发首选**，精度更高 |

---

## 2. 代码书写规范 (考试重灾区) 
这是手写代码和改错题最容易扣分的地方！
### 规则一：后缀 (Suffix)
* **float**：数字后面**必须**加 `f` 或 `F`。
* **double**：数字后面**可以**加 `d` 或 `D`，也可以**不加**（<span style="background:#d2cbff">默认就是 double</span>）。

### 规则二：科学计数法
* 使用 `E` 或 `e` <span style="background:rgba(236, 57, 57, 0.55)">表示 10 的多少次方</span>。
* 例如：`3.14E2` 等于 $3.14 \times 10^2 = 314.0$。

### 考试防坑实例 (根据你的错题整理)
```java
// ❌ 错误示范 (这也是考试改错题常见考点)
float f1 = 3.14;    // 💥 编译报错！3.14 是 double，放不进 float
float f2 = 11.1;    // 💥 编译报错！

// ✅ 正确示范
float f3 = 3.14f;   // OK！加了 f，告诉编译器这是 float
float f4 = 3.14F;   // OK！大写 F 也可以
double d1 = 3.14;   // OK！默认就是 double
double d2 = 3.14d;  // OK！显式声明，虽然没必要
double d3 = 1.23E10;// OK！科学计数法
```
## 3. 浮点数的精度陷阱 (高分加分项) 

> **注意**：浮点数在计算机中是“近似值”，不能用于精确计算（如金融金额）。

```Java
double a = 0.1;
double b = 0.2;
System.out.println(a + b); 
// 😱 输出不是 0.3，而是 0.30000000000000004
// 考试如果考程序输出，千万别想当然！
```

# 字符型 (char)

## 1. 字符型核心特征 
> **定义**：用来存**单个**字符。
> **本质**：Java 使用 **Unicode** 编码（万国码），所以 `char` 能存汉字！

| 特征 | 说明 | 实例 |
| :--- | :--- | :--- |
| **符号** | 必须用 **单引号** `' '` | `char c = 'A';` |
| **大小** | 2 字节 (16 bit) | `char c = '中';` (汉字也是2字节) |
| **范围** | $0 \sim 65535$ | 无符号整数 (没有负数) |

### 考试易错点 (单引号 vs 双引号)
* `'A'` $\rightarrow$ 是 **char** (基本类型)。
* `"A"` $\rightarrow$ 是 **String** (引用类型，其实是一个对象)。
* `char c = "A";` $\rightarrow$ ❌ **编译错误** (类型不兼容)。

---

## 2. ASCII 码与运算 (必背数值) 
> `char` 在计算机里就是<font color="#ff0000">整数</font>。考试考“推算”时，只需记住三个基准点。
* **`'0'`** = **48**
* **`'A'`** = **65** (大写)
* **`'a'`** = **97** (小写，比大写大 32)
### 考试防坑实例 (隐式转换)
```java
char c1 = 'a'; // 97
char c2 = 'b'; // 98

// 坑点 1：char 参与数学运算变 int
System.out.println(c1 + 1);   // 输出 98 (97+1)，而不是 'b'
System.out.println(c1 + c2);  // 输出 195 (97+98) -> 刚才的习题 C

// 坑点 2：字符串连接
System.out.println("Result: " + c1 + 1); // 输出 "Result: a1" (变成了字符串拼接)
```

## 3. 转义字符 (Escape Sequences) 🏃

> 用反斜杠 `\` 开头，改变字符原有的含义。

|**符号**|**含义**|**记忆法**|
|---|---|---|
|`\n`|换行 (New line)|**n**ew line|
|`\t`|制表符 (Tab)|**t**ab|
|`\\`|打印一个反斜杠 `\`|自己转义自己|
|`\"`|打印一个双引号 `"`|在字符串里打印引号|
|`\'`|打印一个单引号 `'`|在字符里打印引号|

### 考试防坑实例

```Java
// 问：下面这行代码输出什么？
System.out.println("ab\tcd");
// 答：ab    cd (中间有大空格)

// 问：如何打印出路径 C:\Windows ？
// ❌ 错误：String path = "C:\Windows"; (\W 没定义，或者是转义了W)
// ✅ 正确：String path = "C:\\Windows"; (必须双写斜杠)
```

# 布尔与字符串 

## 1. 布尔类型 (Boolean) 
> **核心定义**：专门用来控制逻辑流程（对/错，真/假）。
> **内存**：理论上占 1 bit，但实际内存占用取决于虚拟机实现（通常不用纠结这个，考试不考）。

* **取值范围**：只有两个值！**`true`** (真) 和 **`false`** (假)。
* **关键字**：`boolean` (<span style="background:rgba(236, 57, 57, 0.55)">注意全是小写</span>)。

### 考试防坑实例 (C语言后遗症) 
Java 的布尔非常“死板”，**不接受**任何数字或字符串替代！

```java
boolean isVip = true;   // ✅ 正确：标准写法
boolean isOver = false; // ✅ 正确

// ❌ 经典错误示范 (改错题常客)
boolean b1 = 1;      // 💥 错！Java里 1 不是 true (C语言可以，Java不行)
boolean b2 = 0;      // 💥 错！
boolean b3 = "true"; // 💥 错！这是字符串，不是布尔值
boolean b4 = '真';   // 💥 错！
```
## 2. 字符串类型 (String) 

> 核心定义：一串字符的集合。
> 
> 本质：不是基本数据类型，它是<span style="background:rgba(163, 67, 31, 0.2)"> 引用数据类型</span> (Reference Type)。

- **符号**：必须使用 **双引号** `""` 括起来。
    
- **类名**：`String` (注意 **S** 必须大写，因为它是一个<span style="background:#ff4d4f">类</span>)。

### ⚔️ 字符(char) vs 字符串(String)

这是选择题里最容易混淆的概念，一定要分清！

|**特性**|**char (字符)**|**String (字符串)**|
|---|---|---|
|**类型**|基本数据类型 (Primitive)|**引用数据类型** (Reference)|
|**符号**|**单引号** `' '`|**双引号** `" "`|
|**容量**|只能存 **1个** 字符|可以存 **0个到无数个** 字符|
|**示例**|`'A'`, `'中'`|`"A"`, `"中国"`, `"Hello World"`|
|**运算**|可以加减 (变int)|只能拼接 (Connect)|

### 考试防坑实例

Java

```Java

String s1 = "Hello";  // ✅ 正确
String s2 = "";       // ✅ 正确：空字符串
String s3 = "A";      // ✅ 正确：虽然只有1个字，但用了双引号就是String

char c1 = "A";        // ❌ 错误：char 不能用双引号
String s4 = 'A';      // ❌ 错误：String 不能用单引号
```

# 价格该用什么存？ (番外篇)

## 1. 为什么考试选 double？ (The Exam Way) 
> **现状**：考试题目通常很简单，不会涉及复杂的金融计算。
> **选择逻辑**：
> 1.  价格通常有小数 (如 9.9 元)。
> 2.  **`int` 不行**：`int` 只能存整数，`9.9` 会变成 `9`，亏钱了！
> 3.  **`float` 不推荐**：精度太低，且赋值要加 `f` 很麻烦。
> 4.  **`double` 最优解**：书写简单，支持小数。

### ✅ 考试标准写法
```java
// 题目：定义一个商品价格
double price = 19.9; // 考试满分写法
```

## 2. 为什么 double 其实不精准？ (The Truth) 🤯

> **原理**：`double` 是浮点数 (Floating Point)，它是“飘忽不定”的。 计算机内部用二进制存小数，有些小数（比如 0.1）在二进制里是无限循环小数，存不下，只能存个**近似值**。

### 经典“鬼故事” (面试必问)

```Java
double a = 0.1;
double b = 0.2;
System.out.println(a + b); 
// 你的直觉：0.3
// 实际输出：0.30000000000000004 😱
```

> **结论**：如果用 `double` 做电商系统，账目永远对不上！

## 3. 以后工作用什么？ (The Pro Way) 💼

> **类名**：`java.math.BigDecimal` **特点**：它是专门用来做**精确计算**的对象，就像你要算钱必须用算盘，不能用估算。

```Java
// 仅作了解，考试千万别写，写不完的！
BigDecimal p1 = new BigDecimal("0.1");
BigDecimal p2 = new BigDecimal("0.2");
BigDecimal sum = p1.add(p2); // 结果是精确的 0.3
```
